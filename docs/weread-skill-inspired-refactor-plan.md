# 微信读书 Skill 启发的产品改造方案

## 背景

参考文章：[ima接入微信读书Skill，教程来了！防止读完就忘，构建你的阅读知识体系](https://mp.weixin.qq.com/s/TEpBFRcEr3SGAP25ajXO4Q)

文章的核心价值不是“如何接入 ima”，而是把微信读书的书架、笔记和阅读数据变成可被 AI/Skill 调用的长期知识资产。它提供了一个清晰的产品信号：用户真正需要的不是更多数据入口，而是把“读过、划过、想过”的内容整理成可复盘、可提问、可输出、可继续使用的个人知识体系。

这个方向与当前产品定位一致：本地优先管理微信读书资产，通过复盘、阅读指南、选书决策和 Markdown 导出形成闭环。后续改造应吸收文章里的场景表达和任务模板，但不能照搬 ima 的通用 copilot 或 Skill 广场形态。

## 核心判断

当前产品已经具备底层能力：

- 同步和浏览书架、笔记、统计。
- 生成单本 AI 复盘、阅读统计复盘、阅读指南和跨书路线。
- 管理候选书和选书决策。
- 导出笔记、复盘、路线和批量资产。
- 保持本地优先、手动触发 AI、凭据不外泄的边界。

下一步不应继续横向增加 AI 功能，而应把已有能力包装成用户更容易理解和启动的“阅读工作流模板”。也就是从“功能入口”收口为“场景入口”：

- 不是“AI 总结”，而是“把这本书整理成可复盘笔记”。
- 不是“阅读统计复盘”，而是“回顾本月阅读投入和下个月怎么调整”。
- 不是“跨书路线”，而是“围绕一个主题整理多本书的观点差异和下一本阅读顺序”。
- 不是“设置凭据”，而是“连接后能同步书架、整理笔记、生成复盘、导出资产”。

## 改造目标

### 1. 场景模板化

把文章中的高频指令转化为产品内稳定模板。模板只负责描述用户任务、输入范围和输出物，不引入通用聊天框。

首批模板应覆盖：

- 单本整理：按“核心观点 / 代表摘录 / 我的关注 / 行动项 / 复盘问题”整理一本书。
- 主题对比：围绕一个主题，对比多本书或多个作者的观点。
- 阅读复盘：基于周期统计总结阅读节奏、偏好变化和下一步行动。
- 选书取舍：基于本地候选书解释下一本为什么现在读、其他书为什么暂缓。
- 输出打包：把笔记、复盘、行动清单和路线导出为 Markdown 资产。

### 2. 配置流程价值化

设置页不只展示技术配置，而要明确连接后的可用成果：

- 微信读书凭据：用于同步书架、笔记、统计和打开微信读书原始入口。
- AI Provider：仅在用户手动生成复盘、指南、统计解读或选书决策时使用。
- 本地数据目录：用于保存缓存、AI 输出、反馈状态、导出记录和备份。

配置成功后的主动作应直接指向下一步，例如同步书架、同步笔记、生成第一份复盘或保存候选书。

### 3. 输出物体系收口

文章强调“读 -> 记 -> 问 -> 用”的闭环。对应到本产品，最终价值应落在可保存、可导出、可迁移的资产上。

需要统一说明和体验：

- 每类 AI 输出的输入范围。
- 每类输出是否可导出 Markdown。
- 生成结果如何缓存和刷新。
- 导出内容如何标记来源、生成时间和数据边界。
- 哪些内容不会被默认发送给 AI。

### 4. 保持产品边界

本次改造不改变产品定位：

- 不做泛 AI 聊天。
- 不做全书内容问答。
- 不做通用知识库或向量检索。
- 不自动后台生成 AI 内容。
- 不把微信读书数据写回远端。
- 不为了模板化新增复杂字段、任务系统或插件市场。

## 页面改造建议

### 总览页

目标：让用户打开应用后先看到“今天可以完成什么阅读资产动作”。

建议：

- 保留已有“今日可做”动作池，不新增新的大区块。
- 动作文案从功能名转为场景名，例如“整理最近新增笔记”“回顾本月阅读投入”“为候选书做下一本取舍”。
- 每个动作补充来源说明，例如来自本地笔记缓存、已生成复盘、本地候选书或统计缓存。
- 无数据时仍保持单一主动作，不同时暴露多个入口。

验收标准：

- 首屏最多展示 5 个主动作。
- 每个动作都能进入明确页面或触发明确本地操作。
- 不因为模板化而自动调用 AI 或自动同步远端数据。

### 设置页

目标：让配置流程表达“连接后可以得到什么”，而不只是填写字段。

建议：

- 微信读书凭据卡片增加连接后能力说明：同步书架、读取笔记、统计复盘、导出资产。
- AI Provider 卡片增加使用边界说明：手动生成时才调用，不读取或显示已保存 Key 明文。
- 凭据配置成功后提供下一步按钮：同步书架、同步笔记或进入复盘中心。
- 错误态给出恢复路径，不暴露 API Key、数据库路径和原始请求体。

验收标准：

- 新用户能理解为什么要配置两个凭据。
- AI 未配置时，生成入口只引导设置，不自动重试。
- 配置页不引入新的后台任务或默认上传行为。

### 复盘中心

目标：把已有 AI 能力包装为可理解的整理场景。

建议：

- 在书籍复盘、阅读报告、阅读指南之间增加统一的“可生成资产”表达。
- 增加轻量模板入口：
  - “整理一本书”
  - “回顾一段时间”
  - “规划当前书”
  - “决定下一本”
- 模板入口只跳转到已有页面和已有生成流程，不新增通用输入框。
- 对“数据不足”的书籍明确提示先同步笔记或继续阅读。

验收标准：

- 模板入口不复制已有功能逻辑，只复用现有页面能力。
- 每个模板都对应行动、决策或交付物之一。
- 复盘结果仍能导出 Markdown 或复制为结构化文本。

### 单本书详情和单本笔记

目标：让单本书从“数据展示”转为“整理入口”。

建议：

- 详情页本书管理区使用场景文案：查看笔记、整理成复盘、生成阅读指南、找相似书、加入候选。
- 单本笔记页在 AI 入口前展示输入规模：划线数、想法数、章节覆盖和数据边界。
- “AI 总结”文案可逐步替换为“整理成复盘”，避免用户误解为泛总结。
- 代表性摘录必须来自用户提供的划线或想法原文，不改写为书中原文。

验收标准：

- 用户进入单本书后能判断下一步是继续读、看笔记、生成复盘还是导出。
- 非电子书仍保持轻管理边界，不伪造完整详情。
- 单本复盘不发送其他书籍笔记或全量书架。

### 候选书架和选书决策

目标：把“推荐下一本”明确为本地候选取舍，而不是泛推荐。

建议：

- 候选书架空态继续引导发现页保存候选。
- 选书决策模板强调输入范围：只基于本地候选、可选当前书、可选统计信号和已生成摘要。
- 结果页强化三类输出：推荐当前下一本、暂缓原因、具体阅读动作。
- 从阅读指南结果页提供跳转到候选选择的路径，形成“当前书怎么读 -> 下一本读什么”的闭环。

验收标准：

- 没有候选书时不展示“推荐下一本”主动作。
- 选书决策不会读取全量书架、同步远端笔记或发送原始笔记正文。
- 结果能导出或复制为决策记录。

### 导出和资产说明

目标：把“知识资产”落到文件和可迁移内容上。

建议：

- 统一导出说明：来源、生成时间、输入范围、是否包含 AI 内容。
- 统一 Markdown front matter 的关键字段：书籍、作者、范围、生成时间、来源类型、数据边界。
- 导出失败保留用户选择范围，并提供重试或更换目录。
- 批量导出不自动生成缺失复盘，只导出已有资产。

验收标准：

- 导出文件不包含 API Key、数据库路径、原始接口响应或敏感日志。
- 导出内容能被外部 Markdown 工具直接归档。
- 批量导出行为不触发远端同步或 AI 生成。

## 分阶段计划

### P0：文案和入口重命名

目标：低风险验证“场景模板化”是否提升理解，不改变数据结构和调用链。

范围：

- 梳理页面上的 AI 入口文案。
- 设置页增加连接后能力说明。
- 复盘中心增加模板式入口文案，但跳转到现有页面。
- 总览动作标题改为场景表达，保留现有排序和去重逻辑。

不做：

- 新增 AI prompt。
- 新增数据库表。
- 新增通用聊天框。
- 自动生成或后台同步。

验收标准：

- 现有测试不因文案外的行为变化失败。
- 新文案能解释输入范围和输出物。
- 用户仍能通过原有路径完成同步、生成、导出。

### P1：模板配置常量化

目标：避免各页面散落重复模板文案，保持 DRY。

范围：

- 新增一个轻量模板配置模块，集中维护模板名称、适用页面、输入范围、输出物和目标路由。
- 总览、复盘中心、设置页只消费模板配置，不复制业务规则。
- 为模板配置补单元测试，覆盖可见性、路由目标和边界说明。

设计约束：

- 模板配置只描述 UI 和路由，不承担 AI prompt 拼接。
- 业务判断仍保留在现有页面状态和 helper 函数中。
- 不为了模板引入新的全局状态管理。

### P2：输出物说明统一

目标：统一复盘、路线、统计报告和批量导出的资产边界。

范围：

- 统一导出元信息。
- 统一数据边界说明组件。
- 统一失败恢复文案。
- 检查 Markdown 导出是否存在重复实现，可在已有导出模块内小步收敛。

验收标准：

- 每类导出都能说明来源和生成方式。
- 失败态不丢失上下文。
- 不把不同导出强行抽象成一个胖接口。

### P3：外部 Skill 形态预研

目标：评估是否把本地能力暴露为外部可调用的 Skill/MCP 形态。

进入条件：

- 本地数据边界和导出体系稳定。
- 主要阅读资产已经有清晰缓存和版本说明。
- 用户确实需要从外部 Agent 调用本地阅读资产。

预研范围：

- 只读能力优先：列书架摘要、列已有复盘、导出已有资产。
- 敏感能力需显式确认：读取笔记正文、调用 AI、批量导出。
- 外部调用默认不返回凭据、数据库路径和原始接口响应。

暂不进入开发：

- Skill 广场。
- 插件市场。
- 多账号共享。
- 远端知识库同步。

## 风险和控制

### 风险 1：模板化变成伪聊天

控制：

- 模板只映射到现有页面和已有生成流程。
- 不提供自由聊天输入框。
- 不支持用户要求任意问题都能回答。

### 风险 2：AI 能力边界被稀释

控制：

- 每个模板必须明确输入范围。
- 所有 AI 调用继续由用户手动触发。
- 不在进入页面时自动调用 AI。

### 风险 3：文案重复和维护成本上升

控制：

- P1 将模板配置常量化。
- 页面只消费模板，不复制长文案。
- 保持模板数量少，首批不超过 5 个。

### 风险 4：导出资产语义不一致

控制：

- P2 统一导出元信息和数据边界说明。
- 不强行合并不同业务导出，只统一公共字段和失败体验。

## 推荐执行顺序

1. 先落 P0：调整设置页、总览、复盘中心和单本入口文案。
2. 再落 P1：把模板配置抽出，补轻量测试。
3. 然后落 P2：统一导出资产说明和失败恢复。
4. 最后评估 P3：是否需要外部 Skill/MCP 形态。

这个顺序符合 KISS 和 YAGNI：先用低风险文案和入口验证理解，再抽象复用；只有当外部调用需求明确时，才扩展 Skill 形态。
