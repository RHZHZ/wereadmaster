# 复盘页阅读时间轴多周期可视化改造方案

## 结论

复盘页“阅读时间轴 / 按阶段看阅读变化”不应该用同一种图表覆盖所有周期。

这个模块的核心任务是解释阅读投入的变化：在不同周期下，用户关心的问题不同，图表也应该跟着切换。

- 周度：看 7 天内哪天投入最高，适合柱状图。
- 月度：看一整月内的起伏、高峰和回落，适合面积折线图。
- 年度：看 12 个月之间的投入差异，适合月度柱状图加平均线。
- 总计：看多年长期趋势，适合年度面积趋势图。

热力图不再作为阅读报告页时间轴主图。它更适合留在统计页，作为“日期分布 / 连续性”的表达。

## 当前问题

1. 热力图和复盘页语义不完全匹配

- “按阶段看阅读变化”强调趋势、波动和阶段差异。
- 热力图强调日期分布、活跃天数和连续性。
- 在复盘页里，用户更需要判断“变化原因”，而不是只看“哪天读了”。

2. 热力图难以填满宽面板

- 热力图是 7 列日历矩阵。
- 日期格需要接近正方形，不能无限横向拉伸。
- 在宽卡片中会出现明显右侧空白。
- 如果强行放大日期格，会让图表像按钮面板，信息密度下降。

3. 当前条形列表在月度模式过长

- 月度有十几个分桶时，列表会占用大量垂直空间。
- 用户需要逐行扫读，难以看出整体走势。

4. 四个周期的数据粒度不同

- 周度通常是 7 个日分桶。
- 月度通常是 28-31 个日分桶，其中有效分桶更少。
- 年度通常是 12 个月分桶。
- 总计通常是多年分桶。

同一种图表无法同时适配这些数据粒度。

## 设计目标

1. 周期语义清晰

- 每个周期使用最适合该粒度的图表。
- 不为了视觉统一牺牲可读性。

2. 横向空间充分利用

- 宽面板内主图应能横向铺开。
- 避免热力图左侧小块、右侧大空白的问题。

3. 保留复盘解释能力

- 主图只负责展示趋势。
- 阶段总结、AI 对照、代表主题继续承担解释。

4. 控制改造范围

- 只改复盘页阅读时间轴模块。
- 不改统计页主图逻辑。
- 不改全局滚动条。
- 不改书籍详情页。

## 周期图表方案

### 周度：7 日柱状图

适合问题：

- 本周哪天读得最多？
- 哪几天没有投入？
- 周末和工作日是否有差异？

视觉形态：

```text
阅读时间轴                         5 个分桶
按阶段看阅读变化

周一  周二  周三  周四  周五  周六  周日
▁     ▆     ▂     ▁     █     ▃     ▅

高峰：周五 48 分钟
平均：18 分钟/有效日
```

建议：

- 使用柱状图，不使用折线。
- 柱子可以显示完整 7 天，包括 0 值。
- 高峰日突出显示。
- hover / focus 显示具体日期和时长。

### 月度：面积折线图

适合问题：

- 本月阅读投入是上升还是下降？
- 哪些阶段明显波动？
- 高峰之后是否回落？
- 是否存在持续低谷？

视觉形态：

```text
分钟
50 |        ● 高峰 52 分钟
40 |                 ●
30 |           ●
20 |   ●
10 |      ●       ●      ●
 0 +--------------------------------
     2/1  2/5  2/10 2/15 2/20 2/25
```

建议：

- 使用面积折线图作为主图。
- 横向铺满面板宽度。
- X 轴显示关键日期刻度，不显示所有日期。
- Y 轴以分钟为单位。
- 高峰点加标记。
- 可选展示平均线。

### 年度：12 月柱状图 + 平均线

适合问题：

- 哪个月投入最多？
- 阅读是否有明显淡旺季？
- 当前月份是否高于全年平均？

视觉形态：

```text
1月 2月 3月 4月 5月 6月 ... 12月
▃   █   ▅   ▁   ▆   ▂      ▄
---------------- 平均线
```

建议：

- 使用 12 月柱状图。
- 不以折线为主，因为月份是离散单位。
- 加一条平均线帮助判断高低。
- 最高月和当前月可以突出显示。

### 总计：年度面积趋势图

适合问题：

- 长期阅读投入是否增长？
- 哪一年是高峰？
- 最近一年是否回落？
- 阅读资产是否持续积累？

视觉形态：

```text
2021   2022   2023   2024   2025
▁▁▃▃▅▅████▆▆
```

建议：

- 使用年度趋势面积图。
- 如果只有 1-2 年数据，退化成年度柱状图。
- 高峰年标记。
- 可选展示近一年同比变化。

## 信息层级

每个周期图表都应保持同一套信息结构：

```text
[标题区]
阅读时间轴                         N 个分桶
按阶段看阅读变化

[主图]
按周期切换图表

[摘要指标]
有效分桶数 / 高峰 / 平均 / 总时长，报告页左栏采用 2x2 排布，避免长时长文本被截断

[阶段总结]
稳定段 / 波动段 / 低谷段

[AI 对照]
用 AI 生成的节奏判断补充解释
```

图表不应该承载所有解释。它只负责让用户快速看出变化，再由下方阶段卡片解释原因。

## 组件规划

### 新增复盘时间轴图表组件

建议新增：

- `src/features/reading-review/components/ReviewTimelineChart.tsx`

职责：

- 根据 `mode` 选择图表形态。
- 接收 `buckets: ReadingTimeBucket[]`。
- 输出统一的时间轴图表区域。
- 处理空状态。

接口草案：

```ts
type ReviewTimelineChartProps = {
  mode: ReadingStatsMode;
  buckets: ReadingTimeBucket[];
};
```

内部拆分：

```text
ReviewTimelineChart
  ├─ WeeklyBarTimeline
  ├─ MonthlyAreaTimeline
  ├─ AnnualBarTimeline
  └─ OverallAreaTimeline
```

### 保留 `ReviewTimeSegments`

位置：

- `src/features/reading-review/components/ReviewTimelineSection.tsx`

保留内容：

- 阶段总结。
- AI 对照。
- 代表主题。
- 补充节奏提示。

`ReviewTimelineSection` 只负责组合：

```tsx
<ReviewPanelHeading />
<ReviewTimelineChart mode={mode} buckets={buckets} />
<ReviewTimeSegments ... />
```

### 不复用 `ReadingHeatmap` 作为主图

原因：

- `ReadingHeatmap` 是日历分布组件。
- 复盘页主任务是趋势解释。
- 热力图可继续留在统计页，或后续作为月度辅助视图。

## 数据处理规则

1. 有效值

- `readTimeSeconds <= 0` 参与坐标轴占位，但不计入有效分桶。
- tooltip 仍可显示 0 分钟。

2. 时间排序

- 所有图表按 `startTime` 升序展示。

3. 单位

- 主轴使用分钟。
- tooltip 使用 `formatDuration`。

4. 高峰

- 每个周期都计算最高 `readTimeSeconds`。
- 高峰点 / 高峰柱统一突出。

5. 平均

- 周度、月度可展示有效分桶平均值。
- 年度建议展示全年平均月投入。
- 总计可展示年度平均。

## 样式原则

1. 图表应填满宽面板

- 折线图和柱状图都应使用可伸缩宽度。
- 不使用固定 320px / 360px 的窄图。

2. 图表高度保持克制

- 周度：在报告页左栏约 180px。
- 月度：在报告页左栏约 220px。
- 年度：在报告页左栏约 180px。
- 总计：在报告页左栏约 220px。

3. 与复盘页视觉统一

- 使用现有 `var(--green)`、`var(--gold)`、`var(--line)`。
- 不新增强装饰。
- 不引入全局主题变量。

4. 移动端

- 图表横向仍铺满容器。
- X 轴标签减少。
- tooltip 不溢出屏幕。

## 实施步骤

1. 回退热力图作为复盘页主图的实现

- `ReviewTimelineSection` 不再在 monthly 下渲染 `ReadingHeatmap`。
- 如果 `ReadingHeatmap` 已增加 `showHeading`，可保留，不影响统计页。
- 移除 `.review-timeline-panel .reading-heatmap` 的局部样式。

2. 新增 `ReviewTimelineChart`

- 先实现周度柱状图、月度面积折线图、年度柱状图、总计面积图。
- 所有图表共享数据格式化和峰值计算。

3. 替换 `ReviewTimeline`

- 在 `ReviewTimelineSection` 中用 `ReviewTimelineChart` 替代旧列表。
- 保留旧列表逻辑作为必要时的 fallback，或迁入新组件内部。

4. 调整样式

- 新增 `.review-timeline-chart` 相关样式。
- 只限定在复盘页时间轴模块。
- 不改统计页样式。

5. 验证

- 周度展示 7 日柱状图。
- 月度展示面积折线图。
- 年度展示 12 月柱状图和平均线。
- 总计展示年度面积趋势。
- 阶段总结仍在图表之后。
- `npm run build` 通过。
- 复盘页真实数据截图检查。

## 验收标准

1. 周度

- 7 天结构清晰。
- 最高日可识别。
- 0 值日期不造成布局断裂。

2. 月度

- 折线横向铺满。
- 高峰日可识别。
- 趋势起伏比热力图更明显。

3. 年度

- 12 个月柱状清晰。
- 平均线可辅助判断高低。
- 最高月可识别。

4. 总计

- 年度长期趋势清晰。
- 数据年份较少时有合理 fallback。

5. 范围

- 不修改全局滚动条。
- 不修改书籍详情页。
- 不重做统计页。
- 不新增图表库。

## 不做事项

- 不把热力图作为复盘页主图。
- 不为四个周期强行统一一种图表。
- 不修改阅读统计接口。
- 不改 AI 复盘生成逻辑。
- 不引入 Canvas / SVG 大型图表库。
- 不做无关页面视觉调整。
