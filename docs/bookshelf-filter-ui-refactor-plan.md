# 书架筛选区 UI 改造规划

## 背景

当前书架页顶部筛选区同时承担两类任务：

- 内容类型筛选：`全部 / 电子书 / 有声书 / 文章收藏`。
- 父分类筛选：`全部分类 / 文学 / 计算机 / 艺术 / 科学技术...`。

这两组控制目前都以大号胶囊按钮呈现，视觉层级接近，容易让用户误以为它们是同一组菜单，或者误解为“二级菜单”。实际数据语义上，第一组来自书架条目类型，第二组来自 `entry.category` 的父分类聚合，它们不是同一层导航。

本轮改造目标是让用户清楚理解：先按内容类型收口，再在当前类型下按分类细分。

## 目标

1. 明确区分“类型”和“分类”的视觉层级。
2. 保持当前书架数据口径不变：`book / album / mp` 仍是唯一一级类型。
3. 分类筛选跟随当前类型变化，避免在 `文章收藏` 等场景展示无意义分类。
4. 降低顶部筛选区占用高度，减少首屏被筛选 chip 挤占。
5. 为未来区分 `导入书 / 公众号类书籍` 预留位置，但本轮不实现来源子类型。

## 非目标

- 不新增后端字段。
- 不修改 `/shelf/sync` 的解析口径。
- 不把 `导入书 / 公众号类书籍` 提升为一级类型。
- 不重做书架卡片、同步流程、候选保存和详情入口。
- 不引入新的 UI 组件库。

## 信息架构

推荐结构：

```text
类型：全部 | 电子书 | 有声书 | 文章收藏
分类：全部 | 文学 | 计算机 | 艺术 | 科学技术 | 更多
```

说明：

- `类型` 是主筛选，控制当前书架条目集合。
- `分类` 是从属筛选，只在当前类型集合内生效。
- `搜索` 继续作为关键词过滤，不改变分类 chip 的可选集合，避免用户输入时分类栏频繁跳动。
- `全部` 在类型行表示全部条目，在分类行表示当前类型下的全部分类。

## 推荐 UI

### 桌面端

- 类型行使用分段控件样式，位置靠近书架标题和搜索区域。
- 分类行使用更轻的 chip 样式，并加弱标签 `分类`。
- `更多` 紧跟分类 chip，不远离主控制区。
- 展开后分类可以换行展示，收起时只保留高频分类。

建议布局：

```text
书架标题 / 同步按钮
统计摘要

筛选区
类型  [全部] [电子书] [有声书] [文章收藏]
分类  [全部] [文学 218] [计算机 113] [艺术 33] ... [更多 4]

搜索框
书架网格
```

### 窄屏端

- 类型行保持横向滚动或紧凑换行。
- 分类行默认横向滚动，避免撑高页面。
- `更多` 仍作为分类行最后一个控制，不单独占一整行。

## 交互规则

### 类型切换

- 点击 `全部 / 电子书 / 有声书 / 文章收藏` 后，立即更新书架列表。
- 分类选项基于当前类型重新计算。
- 如果当前分类在新类型下不存在，自动回到分类 `全部`。
- `文章收藏` 只有一个入口时，隐藏分类行。

### 分类筛选

- 分类统计只统计当前类型下的条目。
- 分类不跟随搜索关键词变化，避免搜索时筛选项闪烁。
- 默认展示高频分类，低频分类通过 `更多` 展开。
- 展开状态只影响分类行，不影响类型行。

### 搜索

- 搜索继续匹配书名、作者、分类。
- 搜索结果在当前 `类型 + 分类` 范围内过滤。
- 清空搜索不重置类型和分类。

### 空态

- 当前类型无条目：提示切换类型或同步书架。
- 当前分类无条目：提示查看当前类型下全部分类。
- 搜索无结果：提示清空搜索或更换关键词。

## 数据口径

当前一级类型保持：

```ts
type ShelfEntryType = "book" | "album" | "mp";
```

分类来源保持：

```ts
entry.category
```

父分类规则保持当前实现，即将 `计算机-*` 等子分类聚合为一级分类。

未来如果要区分导入书和公众号类书籍，建议新增可选子类型，而不是修改一级类型：

```ts
type BookSourceType = "normal" | "imported" | "mpBook" | "unknown";
```

该字段只有在 `/shelf/sync` 或本地详情缓存中能稳定识别时才展示；本轮不实现。

## 实施步骤

### 1. 调整派生数据

- 将分类选项从“全量书架条目”改为“当前类型条目”。
- 保持分类统计不受搜索关键词影响。
- 在类型变更后处理无效分类，回退到 `all`。

验收：

- 选择 `电子书` 时，分类数量只来自电子书。
- 选择 `文章收藏` 时，不展示无意义分类行。

### 2. 调整 JSX 结构

- 将现有 `filter-tabs` 拆成语义清晰的筛选面板。
- 类型行使用 `aria-label="书架类型筛选"`。
- 分类行使用 `aria-label="书架分类筛选"`。
- 保留按钮语义和键盘可达性。

验收：

- 读屏和 DOM 结构能区分类型筛选与分类筛选。
- 原有筛选功能不回退。

### 3. 调整样式

- 类型行使用更强的选中态。
- 分类行使用较轻 chip，字号和高度低于类型行。
- 移动端控制高度，避免三行以上筛选长期占据首屏。
- 暗色主题同步覆盖新 class。

验收：

- 第一眼能看出类型是主筛选，分类是从属筛选。
- 宽屏和窄屏都没有文字溢出、控件重叠或布局跳动。

### 4. 验证

需要覆盖：

- `全部 -> 电子书 -> 有声书 -> 文章收藏` 类型切换。
- 分类切换、展开更多、收起分类。
- 类型切换后分类回退。
- 搜索与类型、分类组合过滤。
- 空态和暗色主题。

建议命令：

```bash
npm test -- --run
npm run build
```

如本地启动前端，还需要截图验证桌面端和窄屏端筛选区。

## 工程原则

- KISS：只调整筛选区结构和派生数据，不重写书架页。
- YAGNI：不提前实现导入书、公众号类书籍子类型。
- DRY：复用现有 `filterEntries`、`getParentCategory` 和分类统计逻辑。
- SOLID：保持数据解析、筛选派生、UI 展示各自职责清晰。

## 风险

- 如果分类跟随类型变化，用户切换类型时当前分类可能被重置，需要明确反馈在选中态上。
- 有声书分类字段可能不完整，分类行需要允许隐藏。
- 未来新增来源子类型时，不能把来源和分类混进同一行，否则会再次造成筛选语义混乱。

## 推荐落地顺序

1. 先完成当前筛选区 UI 层级和分类跟随类型。
2. 再观察真实书架数据中导入书、公众号类书籍是否有稳定字段。
3. 字段可靠后，再评估是否增加折叠的 `来源` 筛选，不默认常驻第三行。
