# AI 阅读助手 P2c：候选书源确认状态设计文档

## 背景

P2a/P2b 已完成：

- AI 推荐卡片可搜索微信读书。
- 用户手动选择搜索结果后，可用真实 `bookId` 加入本地候选。
- 搜索失败或无结果时，仍可保存为 `ai-rec-*` 本地候选。

因此候选书架里会同时存在两类 AI 推荐候选：

- `ai-rec-*`：AI 推荐但未确认微信读书书源。
- 真实 `bookId` + 备注包含“已通过微信读书搜索确认”：AI 推荐且已确认微信读书书源。

P2c 的目标是让候选书架清楚表达这两类状态，并允许用户对单本未确认候选手动搜索确认。

## 设计目标

- 候选书架显示候选来源与确认状态。
- 候选书架提供确认状态统计与筛选。
- 筛选或搜索无结果时给出明确原因和恢复入口。
- 未确认 `ai-rec-*` 不再打开无效书籍详情页。
- 未确认候选可在候选书架内搜索微信读书并手动选择匹配项。
- 用户确认后，用真实 `bookId` 替换原 `ai-rec-*` 本地候选。
- 不新增数据库字段。
- 不批量迁移、不自动合并历史候选。
- 不引入微信读书远端写入。

## 非目标

本次不做：

- 不自动把 `ai-rec-*` 替换成真实 `bookId`。
- 不批量删除重复候选。
- 不新增候选确认状态表。
- 不自动选择搜索结果。
- 不做 ISBN、出版社、译者级别匹配。

## 状态推导

状态只从现有数据推导：

```ts
book.localType === "candidate"
book.bookId.startsWith("ai-rec-")
book.localNote?.includes("已通过微信读书搜索确认")
```

### 状态映射

| 条件 | 标签 | 可打开详情 |
| --- | --- | --- |
| `candidate` + `bookId` 为 `ai-rec-*` | AI 推荐 · 未确认书源 | 否 |
| `candidate` + 备注含“已通过微信读书搜索确认” | AI 推荐 · 微信读书已确认 | 是 |
| `candidate` + 真实 `bookId` | 微信读书书目 · 本机候选 | 是 |
| `album` | 有声书 · 轻管理候选 | 否 |
| `mp` | 文章收藏 · 轻管理候选 | 否 |

候选卡片的来源信息从普通文本升级为状态胶囊：

```text
AI 推荐 · 未确认书源
AI 推荐 · 微信读书已确认
微信读书书目 · 本机候选
```

未确认 AI 本地候选禁用详情跳转，但仍保留移除和参与后续候选管理的能力。

选书决策输入弹窗也展示同一套来源状态胶囊。这样用户仍然可以主动把未确认候选纳入决策，但不会误以为它已经对应真实微信读书书目。

候选书架顶部展示确认状态统计：

```text
候选书 / 已确认 / 待确认 / 本地保存
```

候选池支持按来源筛选：

```text
全部 / 已确认 / 待确认 / 轻管理
```

筛选只影响当前页面展示，不改变候选数据，也不影响用户主动选择候选进入选书决策。

筛选后空态按触发原因区分：

| 场景 | 标题 | 操作 |
| --- | --- | --- |
| 仅搜索无结果 | 没有匹配的候选书 | 清空搜索 |
| 仅来源筛选无结果 | 当前筛选下没有候选书 | 显示全部 |
| 搜索 + 来源筛选无结果 | 当前筛选下没有匹配候选 / 当前筛选下没有候选书 | 清空搜索、显示全部 |

空态只恢复当前前端筛选条件，不修改候选数据。

## 原地搜索确认流程

只对“AI 推荐 · 未确认书源”候选展示 `搜索确认`：

```text
点击搜索确认
  -> searchBooks({ keyword: "书名 作者", scope: 0, count: 5 })
  -> 展示最多 5 个微信读书搜索结果
  -> 用户选择结果
  -> 用户确认替换
  -> 先写入真实 bookId 候选
  -> 再移除原 ai-rec-* 候选
```

如果搜索结果已经是本地候选：

- 不覆盖已有候选备注。
- 只移除原 `ai-rec-*` 未确认候选。

如果搜索结果已存在为其它本地阅读状态：

- 不覆盖。
- 不移除原未确认候选。
- 提示用户该书已存在本地阅读状态。

## 边界说明

### 为什么不新增字段

当前已经能从 `bookId` 和 `note` 推导状态。新增字段会带来迁移和兼容成本，不符合 P2c 的轻量目标。

### 为什么只做用户确认后的替换

`ai-rec-*` 可能有用户备注或用户仍想保留。自动合并需要冲突确认、备注合并和删除逻辑，应该单独设计。

本次只在用户明确选择搜索结果并确认替换后，才移除对应的单本未确认候选。

## 验收标准

- 候选书架能区分“AI 推荐未确认”和“AI 推荐已确认”。
- 候选书架可以按“已确认 / 待确认 / 轻管理”筛选候选。
- 搜索或来源筛选无结果时，能清空搜索或切回全部候选。
- `ai-rec-*` 候选不会打开无效书籍详情。
- `ai-rec-*` 候选可以搜索微信读书并手动确认替换。
- 确认替换后使用真实 `bookId` 保存候选。
- 普通微信读书候选仍能打开详情。
- 有声书和文章收藏仍保持轻管理候选标签。
- 类型检查、相关单测、构建通过。
