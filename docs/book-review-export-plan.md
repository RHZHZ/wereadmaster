# 书籍复盘导出计划

## 背景

当前笔记页批量导出已经可以附带导出本地已有的书籍复盘缓存。但继续把复盘导出的筛选、搜索、选择和索引都放进笔记导出向导，会让笔记页承担过多职责。

复盘和笔记的边界不同：

- 笔记导出关注是否有划线/想法、是否需要同步、是否有本地可导出 Markdown。
- 复盘导出关注是否已生成、生成时间、模型版本、复盘内容和来源统计。

因此，复盘导出应归属书籍复盘页。当前已经提供书籍复盘页独立 `导出书籍复盘` 入口，使用步骤弹窗选择已生成复盘、确认导出边界，并生成复盘索引。

## 信息架构

- 笔记页批量导出：主任务是导出划线/想法 Markdown。
- 笔记页批量导出可以继续附带导出同一本书的已生成复盘缓存，但不增加复盘专属控制项。
- 书籍复盘页：独立 `导出书籍复盘` 入口，主任务是导出已生成复盘。
- 书籍复盘页导出弹窗承接复盘搜索、显式选择、设置确认和导出结果，不把批量导出控件常驻堆在列表页。

## 已完成范围

- 只导出已生成复盘缓存。
- 不自动生成缺失复盘。
- 不读取未缓存笔记。
- 支持按用户显式选择范围导出部分复盘，不默认全选。
- 支持在导出弹窗内按书名、作者或复盘概览搜索已生成复盘。
- 支持在已生成复盘列表和导出弹窗内展示本地反馈记录数量。
- 支持在确认步骤选择是否包含行动反馈、复盘问题反馈和代表性摘录，默认全部包含，保持历史导出完整性。
- 生成复盘导出索引，记录书名、作者、缓存更新时间、模型版本和导出文件。
- 后端命令复用 `bookIds` 参数限定导出范围；未选择任何复盘时不能进入导出确认。

## 失败与重试

- 书籍复盘导出失败后仍停留在导出弹窗的结果步骤，不跳回列表页，也不只依赖 toast。
- 失败结果展示错误分类标题、原始错误、恢复建议和状态保留说明。
- 失败后保留已选书籍、搜索范围和导出设置；用户可以直接重试，也可以返回设置或选择范围。
- 失败重试仍只读取已生成复盘缓存，不自动生成缺失复盘，不读取未缓存笔记，不请求微信读书远端。
- 笔记批量导出和书籍复盘导出共用同一套导出失败展示组件，但不共用完整导出向导，避免把两个业务流程耦合。

## 后续扩展

- 评估是否需要继续增加来源统计开关；当前来源统计仍随复盘主体导出。
- 评估是否需要导出后直接打开目录；首版只展示导出路径。
- 评估是否需要压缩包输出；首版仍输出目录。

## 非目标

- 不在导出流程中批量生成复盘。
- 不把复盘专属筛选继续塞进笔记批量导出向导。
- 不做跨书主题整理或语义搜索。
- 不改变单本复盘页已有的单本 Markdown 导出能力。

## 验收标准

- [x] 用户能在书籍复盘页找到独立 `导出书籍复盘` 入口。
- [x] 没有已生成复盘时，导出入口不可用，不自动调用 AI。
- [x] 导出结果只包含已生成缓存，不包含 API Key、数据库路径或原始接口响应。
- [x] 导出结果包含复盘 Markdown 文件和复盘索引。
- [x] 笔记页批量导出仍以笔记 Markdown 为主，复盘只作为兼容附带项。
- [x] 前端支持按用户选择范围导出部分复盘，且不默认全选。
- [x] 导出选择、设置确认和导出结果使用步骤弹窗承载，不挤压书籍复盘主列表。
- [x] 导出设置支持控制行动反馈、复盘问题反馈和代表性摘录是否写入 Markdown，默认全开。
- [x] 导出失败后进入结果步骤，展示分类提示、原始错误、恢复建议，并支持保留状态后重试。
