# 本地书库发布前 QA 清单

## 目标

这份清单用于第二阶段发布前验收，重点确认本地书库和本地阅读器在真实桌面环境中可交付。它不是新功能规划，也不扩大阅读器范围。

验收结论只回答三件事：

- 用户能否稳定导入、阅读、划线、写想法、提问、保存进度和导出本地图书资产。
- 本地版本和微信读书版本是否始终隔离，不发生隐性合并或数据串写。
- 打包后的桌面端是否能在真实数据目录、权限和重启场景下保持一致行为。

## 前置环境

- Windows 桌面端开发机。
- 一份干净 QA 数据目录，避免污染个人真实书库。
- 至少准备以下样本文件：
  - UTF-8 TXT：短文本，带章节标题。
  - UTF-8 TXT：长段落、长 URL、连续英文 token。
  - EPUB：带 OPF `dc:title` / `dc:creator`。
  - EPUB：缺失作者或标题，用于验证文件名 fallback。
  - 破损 EPUB 或无法抽取正文的 EPUB。
  - 不支持格式文件，例如 PDF 或 DOCX。

## 自动化回归

发布前必须全部通过：

```powershell
npx tsc --noEmit --pretty false
npx vitest run src/pages/LocalReaderPage.test.tsx src/pages/LocalLibraryPage.test.tsx
npx playwright test tests/e2e/local-reader.spec.ts
cargo test local_book --lib
npm run build
```

如果 Playwright 内置 `webServer` 退出阶段卡住，可先显式启动 Vite，再让 Playwright 复用该服务；测试结果仍必须显示 `16 passed`。

## 手工验收

### 1. 干净启动

- 使用干净数据目录启动应用。
- 进入书架子菜单，打开本地书库。
- 确认空态是“选择或粘贴本地图书”，没有微信书架数据混入。
- 切换暗色和浅色模式，本地书库和阅读器视觉不破版。

通过标准：本地书库入口清晰，空态可理解，无错误 toast，无控制台运行时错误。

### 2. TXT 导入和阅读

- 导入 UTF-8 TXT。
- 确认卡片显示本地版本、格式、大小和最近阅读信息。
- 打开阅读器，滚动到中后段，返回书库，再次打开。
- 确认进度恢复，书库排序按最近阅读更新。

通过标准：导入、打开、保存进度、恢复进度形成闭环。

### 3. EPUB 导入和阅读

- 导入带元数据 EPUB。
- 确认标题、作者优先来自 OPF。
- 导入缺失元数据 EPUB。
- 确认标题 fallback 到文件名，作者显示可接受的兜底值。
- 打开 EPUB 阅读器，确认展示“保守文本抽取”语义，不出现待接入占位。

通过标准：EPUB 能读纯文本正文，但不承诺精排版。

### 4. 导入失败和重复导入

- 导入不支持格式。
- 导入破损 EPUB。
- 重复导入同一 TXT 或 EPUB。
- 在一次重复导入后直接打开已有记录。

通过标准：失败有明确错误态；重复导入不生成第二张卡；已有记录可继续打开。

### 5. 阅读器选区工具

- 选中一段普通文本。
- 验证浮层在正文附近出现，包含划线、写想法、问 AI、标记、疑问、复制。
- 复制选区，确认剪贴板内容正确。
- 关闭浮层后焦点回到阅读触发点附近。

通过标准：主要输入在正文附近完成，不跑到侧边栏工作台。

### 6. 划线、想法和长文本

- 创建划线。
- 为同一选区写想法。
- 使用长中文、长 URL 和连续英文 token 创建想法。
- 点击侧栏划线卡片和想法卡片，查看详情弹窗。
- 在详情弹窗复制原文和想法，定位回正文。

通过标准：侧栏只显示摘要；完整内容在弹窗查看；长文本不撑开卡片、不横向溢出、不越过视口。

### 7. AI 提问边界

- 未配置 AI Provider 时，选区问 AI。
- 确认只保存草稿或本地提问记录，不展示假回答。
- 配置 AI Provider 后，选区提交一个问题。
- 确认请求只围绕选区；回答进入 AI tab 摘要列表和详情弹窗。

通过标准：AI 不默认渲染在正文分区下，不发送整本书，不展示伪能力。

### 8. 本地和微信版本隔离

- 准备一本与微信书架同名同作者的本地图书。
- 确认只出现疑似版本提示，不自动合并。
- 手动关联后查看本地书库卡片和微信书籍详情页。
- 取消关联后再次查看两边状态。

通过标准：关联只影响展示；不迁移、不复制、不合并笔记、进度、划线或 AI 记录。

### 9. Markdown 导出

- 至少创建一条划线、一条想法和一条 AI 提问记录。
- 导出 Markdown。
- 检查 front matter 包含 `source_kind: local`、`book_id`、`file_hash`、`format`。
- 检查正文不包含微信 note id、微信章节位置或远端同步字段。
- 只保留 AI 提问记录、没有划线和想法时再次导出。

通过标准：导出结构稳定，且始终只包含本地来源内容。

### 10. 重启和打包验证

- 使用开发构建完成一次导入、阅读、标注和导出。
- 关闭应用并重新启动。
- 确认本地书库、阅读进度、划线、想法、AI 记录和偏好仍在。
- 使用打包产物重复 TXT/EPUB 导入、阅读和导出核心路径。

通过标准：开发构建和打包产物行为一致；重启后数据不丢失。

## 失败处理

出现以下任一情况，不应发布：

- 本地书和微信书自动合并。
- 本地导出混入微信笔记或远端字段。
- 导入失败后留下孤儿文件、临时文件或备份目录。
- 正文读取失败时展示上一本文档的标题、浮层或详情弹窗。
- 切书后旧保存请求回写当前书进度状态。
- 未配置 AI 时展示假回答。
- 长文本撑开卡片、遮挡按钮或造成横向滚动。
