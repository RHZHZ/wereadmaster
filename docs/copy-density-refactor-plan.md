# 文案压缩与信息密度微调改造计划

## 背景

每日阅读工作台改造后，页面主路径、资产状态和复盘入口已经更清晰。当前主要问题不是功能冗余，而是多个页面反复解释同一类边界：

- 数据只保存在本机。
- 不会自动同步微信读书。
- 不会自动调用 AI。
- 生成、同步、导出都需要用户手动确认。
- 已生成内容来自本地缓存。

这些说明本身必要，但在标题、副文案、状态条、空态、设置说明和确认弹窗中重复出现，会让页面显得偏重、偏说明书化。此次改造只处理文案和信息密度，不调整页面结构和业务行为。

## 目标

1. 降低高频页面的说明密度，让用户更快判断下一步动作。
2. 保留关键安全边界，但每个页面只在最合适的位置解释一次。
3. 让标题负责表达任务价值，边界条负责表达安全边界，空态负责给下一步动作。
4. 避免继续扩张页面模块，不新增后端命令、不新增自动同步、不新增自动 AI 调用。
5. 保持现有 E2E 和视觉滚动回归稳定。

## 非目标

- 不重做信息架构。
- 不删除主要入口：总览、书架、笔记、统计、复盘、发现、设置都保留。
- 不删除资产状态卡、今日工作台、本地进展等承担方向感的模块。
- 不改变 AI 生成、同步、导出、状态写入的触发方式。
- 不重写设计系统或大面积改 CSS。

## 文案原则

### 用户视角

用户更关心“我现在能做什么、会得到什么、是否需要确认”，不关心页面如何通过前端、Tauri、Rust 或接口参数实现。文案应优先使用任务语言，弱化实现语言。

需要从主界面移除或改写的表达：

- 实现层名词：前端、本地 Rust 层、Tauri 命令、`/readdata/detail`、`searchIdx`、`idx`、`sessionId`。
- 调试和版本细节：`Prompt {version}`、技术信息、内部版本名。
- 改版痕迹：默认引导页已移除、入口已统一、避免重复配置输入。
- 防御性重复：不会自动调用 AI、不会自动同步、不会静默读取、不会发送完整反馈明细。
- 产品黑话：资产底座、本地候选沉淀、沉淀阅读资产、复盘资产。

保留原则：

- 涉及 Key、删除、清除、恢复、迁移、导出范围时，可以保留完整边界说明。
- 涉及 AI 生成时，保留“点击生成才使用哪些输入”，但只出现一次。
- 涉及技术版本时，优先放到详情折叠、复制内容或导出元信息，不在主路径强展示。

### 页面头部

页面头部只回答“这个页面干什么”，不再详细解释本地边界。

建议长度：
- kicker：2-6 个字。
- 标题：8-18 个字。
- 副文案：18-36 个字。

### 边界说明

每个页面最多保留一个显式边界说明区域。高风险页面优先保留：

- AI 生成页。
- 设置页。
- 导出页。
- 清除缓存、迁移数据、恢复备份等危险操作弹窗。

普通信息卡片不重复写“不会自动调用 AI”“不会写回微信读书”。

### 空态

空态只保留：

- 当前为什么为空。
- 下一步做什么。

避免同时解释缓存、同步、生成边界。

### 卡片副文案

卡片副文案只解释当前卡片的作用，不承担完整安全说明。建议控制在 18-28 个汉字。

### 确认弹窗

确认弹窗可以保留完整说明，尤其是删除、清除、迁移、恢复类操作。这里不是压缩重点。

## 删除与替换清单

### 直接删除

这些文案暴露的是开发过程、接口细节或内部实现，用户无法据此完成更好的决策，建议从可见主界面删除。

| 文件 | 当前文案 | 处理 |
| --- | --- | --- |
| `src/pages/BookDecisionPage.tsx` | 默认引导页已移除，生成入口统一放在候选书架，避免在结果页重复配置输入。 | 删除整段。 |
| `src/pages/DiscoveryPage.tsx` | 搜索翻页使用最后一条 searchIdx；相似书翻页使用 idx 和 sessionId。 | 删除整段技术说明。 |
| `src/pages/StatisticsPage.tsx` | 统计同步通过本地 Rust 命令调用 /readdata/detail。 | 删除或替换为“同步后可查看阅读统计”。 |
| `src/pages/ReadingHubPage.tsx` | 不会自动同步、自动生成或打开通用聊天。 | 删除“打开通用聊天”，剩余边界合并到页顶说明。 |
| `src/pages/BookAiSummaryPage.tsx` | 主界面中的 Prompt 版本展示。 | 主界面删除，保留到导出元信息。 |
| `src/pages/ReadingRoutePage.tsx` | 来源版本 + Prompt + 完整反馈明细说明。 | 压缩为更新来源和手动确认。 |

### 替换为用户语言

这些文案表达的边界仍有价值，但当前更像工程说明，建议改成用户可直接理解的动作或影响。

| 当前表达 | 建议表达 |
| --- | --- |
| 前端只调用本地 Tauri 命令。 | 仅通过本机服务读取。 |
| 本地 Rust 层读取凭据。 | 凭据仅在本机使用。 |
| 前端页面不会读取或展示明文。 | 页面不会显示已保存密钥。 |
| 只读取本地 AI 缓存引用。 | 读取已保存的 AI 结果。 |
| 不会静默读取未缓存远端内容。 | 不会自动同步新内容。 |
| 不会自动发送完整反馈明细。 | 生成前可确认输入范围。 |
| 保存 API Key 后开始沉淀阅读资产。 | 保存后可同步书架、笔记和统计。 |
| 本地候选沉淀。 | 保存候选书。 |
| 资产底座。 | 阅读数据基础。 |
| 沉淀阅读资产。 | 整理阅读成果。 |

### 弱化但不完全删除

这些说明有信任价值，但不应占据主路径，可以移到页脚、提示、详情折叠或确认弹窗。

| 场景 | 当前问题 | 建议位置 |
| --- | --- | --- |
| AI Key 安全 | 多处重复“不会读取或展示明文”。 | 设置页 AI 卡片保留一次，其他页面改短。 |
| AI 生成边界 | 头部、边界条、空态重复“不会自动调用”。 | 页面边界条保留一次。 |
| 导出边界 | 导出失败、导出范围和设置里重复说明。 | 导出弹窗保留，失败提示压短。 |
| 本地/微信读书隔离 | 详情页、本地阅读器、设置页均有长说明。 | 来源冲突提示保留，普通卡片压短。 |
| 发现页隐私说明 | 主流程底部解释好友关系和搜索上传。 | 页脚弱提示或帮助说明。 |

## 用户可见术语表

后续改造中优先统一以下术语，避免同一概念在不同页面被叫成多种名字。

| 概念 | 推荐叫法 | 避免叫法 |
| --- | --- | --- |
| 已同步到本机的数据 | 本机缓存 | 资产底座、本地资产库 |
| 已保存但未决定阅读的书 | 候选书 | 本地候选沉淀 |
| 已生成的书籍总结 | 阅读报告 | 复盘资产、AI 资产 |
| 面向下一步阅读的建议 | 阅读指南 | 路线资产、指南资产 |
| 多本书之间的阅读安排 | 阅读路线 | 跨书资产 |
| 已完成的笔记整理 | 整理完成 | 已沉淀 |
| AI 结果版本 | 版本 | Prompt、内部版本 |

## 主要发现

### 1. 设置页说明最密

涉及文件：
- `src/pages/SettingsPage.tsx`

问题：
- 账户、AI、导出、高级维护都反复说明本机保存、不会删除缓存、不会移动 Key。
- 普通卡片和危险确认弹窗承担了相似说明。

处理方向：
- 普通卡片压成一句短说明。
- 危险确认弹窗保留完整影响范围。
- 诊断类说明默认弱化，避免干扰日常设置。

示例：

```text
原：连接微信读书后，才能把书架、笔记和统计沉淀到本地阅读资产库；已保存的 Key 只在本机安全存储中读取。
新：连接后可同步书架、笔记和统计。
```

```text
原：AI 只在你点击生成书籍复盘、阅读指南、统计复盘或选书决策时调用配置的 Provider；不会自动上传书架、其他书笔记或任何 API Key。
新：AI 仅在点击生成时使用确认的输入范围。
```

### 2. 复盘中心重复解释本地缓存

涉及文件：
- `src/pages/ReadingHubPage.tsx`

问题：
- 复盘列表、候选列表、阅读指南库、资产详情、版本详情都在重复“只读本地缓存/不会自动调用 AI”。
- 加载态和空态里也有边界说明，阅读负担偏重。

处理方向：
- 页面顶部保留一次边界说明。
- 加载态只说明“正在读取什么”。
- 空态只说明“从哪里生成后会出现”。
- 更新确认弹窗保留完整边界。

示例：

```text
原：这里只展示已经手动生成过复盘的书，不会自动调用 AI。
新：仅展示已生成复盘。
```

```text
原：这里按书整理已经沉淀下来的指南、路线和复盘资产，方便回看、更新和继续导出。
新：按书归档指南、路线和复盘。
```

### 3. AI 生成页头部和边界条职责重叠

涉及文件：
- `src/pages/BookAiSummaryPage.tsx`
- `src/pages/ReadingRoutePage.tsx`
- `src/pages/BookDecisionPage.tsx`

问题：
- 页面头部、副文案、边界条都在解释“点击生成才发送”。
- 用户真正需要先看到的是生成结果、输入范围和下一步操作。

处理方向：
- 头部讲页面价值。
- 边界条讲输入范围和安全边界。
- 按钮附近只保留必要状态，不重复解释。

示例：

```text
原：优先读取本地持久化缓存；缓存不存在或重新生成时，才会发送当前书的划线和想法。
新：读取缓存；点击生成时使用当前书笔记。
```

```text
原：默认把当前书整理成下一步阅读指南；加入候选书后切换为跨书路线图。不会自动发送笔记，也不会写回微信读书。
新：先生成本书指南；加入候选后生成跨书路线。
```

### 4. 总览有轻微信息重叠

涉及文件：
- `src/pages/DashboardPage.tsx`

问题：
- 总览同时出现“阅读资产”“今日可做”“本地队列”“本地进展”“今日工作台”。
- 模块职责不同，可以保留；但副文案里“本地/同步/资产”重复较多。

处理方向：
- 保留模块结构。
- 压缩本地安全说明。
- 今日工作台动作文案强调结果，不重复讲资产底座。

示例：

```text
原：先把微信读书资产写入本地，再整理复盘、候选和导出内容。
新：先同步书架，再整理复盘和候选。
```

```text
原：只读取本机缓存和本地整理状态，不写回微信读书。
新：来自本机缓存与整理状态。
```

### 5. 书籍详情与单本笔记可压缩辅助说明

涉及文件：
- `src/pages/BookDetailPage.tsx`
- `src/pages/BookNotesPage.tsx`

问题：
- 书籍详情的本地整理、候选入口、资产状态都在讲本机状态。
- 单本笔记导出、随机组和筛选摘要里多次出现“本地已加载”。

处理方向：
- 书籍详情保留资产状态卡，压缩其他说明。
- 单本笔记将“本地已加载”放到导出说明或 tooltip，摘要短一点。

示例：

```text
原：微信读书进度仍以微信读书为准；这里只记录本机的复盘和整理状态，不会写回微信读书。
新：仅记录本机整理状态。
```

```text
原：已从当前筛选条件中随机抽取 3 条笔记，只使用本地已加载内容。
新：已随机抽取 3 条当前笔记。
```

### 6. 发现页底部说明可弱化

涉及文件：
- `src/pages/DiscoveryPage.tsx`

问题：
- “不会读取好友关系/不会上传搜索记录”属于信任说明，但不一定需要在主流程中强展示。

处理方向：
- 主路径保留搜索、推荐、候选沉淀。
- 信任说明移到页脚弱提示，或者收短为一句。

示例：

```text
原：它不会读取好友关系，也不会把搜索记录上传到自建服务器，只服务你的本地候选沉淀。
新：搜索只服务本地候选沉淀。
```

## 执行顺序

### 第一批：高收益压缩

1. `SettingsPage.tsx`
   - 压缩分类 heroDescription。
   - 压缩账户与同步、AI 设置、导出设置普通说明。
   - 保留危险确认弹窗完整说明。

2. `ReadingHubPage.tsx`
   - 压缩复盘中心和阅读指南库头部说明。
   - 压缩加载态、空态。
   - 保留更新确认弹窗的边界说明。

3. `BookAiSummaryPage.tsx` 与 `ReadingRoutePage.tsx`
   - 拆清头部价值文案和边界条文案。
   - 避免同屏重复“不会自动发送”。

### 第二批：高频路径微调

4. `DashboardPage.tsx`
   - 压缩今日动作、本地队列、本地进展副文案。
   - 保留今日工作台结构。

5. `BookDetailPage.tsx` 与 `BookNotesPage.tsx`
   - 压缩本地整理说明、候选入口说明和导出筛选摘要。

6. `DiscoveryPage.tsx`
   - 弱化底部信任说明。
   - 保留候选沉淀路径。

## 验收标准

### 文案标准

- 高频页面同屏不重复出现 2 次以上同义边界说明。
- 普通卡片副文案不超过 28 个汉字，复杂说明转移到边界条或确认弹窗。
- 空态文案控制在 1 句原因 + 1 个动作。
- 页面头部不再承担完整安全说明。

### 行为标准

- 不新增任何自动同步。
- 不新增任何自动 AI 调用。
- 不改变已生成状态、整理状态、候选状态的写入时机。
- 不删除主要入口。

### 回归标准

执行：

```powershell
npm run e2e
npm test
npm run build
```

重点确认：

- 主应用桌面和窄屏滚动视觉回归继续通过。
- 设置弹窗各分类滚动视觉回归继续通过。
- 本地书库和阅读器滚动视觉回归继续通过。
- 无新增水平溢出、遮挡或按钮文本撑出。

## 后续规划方向

这轮只做文案压缩。后续如继续优化，可以分成三个独立方向：

1. 信息层级统一
   - 统一 kicker、标题、副文案、状态条的职责。
   - 建立可复用的页面头部文案规则。

2. 帮助说明折叠
   - 将低频安全说明放进 tooltip、details 或帮助入口。
   - 保留危险操作确认里的完整说明。

3. 术语统一
   - 明确“阅读报告”“统计”“复盘”“阅读指南”“资产”的命名边界。
   - 减少同一概念在不同页面中的近义表达。

4. 技术细节归位
   - 主界面隐藏 Prompt、接口参数和实现层名词。
   - 在导出元信息、调试面板或开发者说明中保留必要追踪信息。

5. 信任说明体系化
   - 将“本机保存、手动生成、手动同步、手动导出”整理成统一的短句库。
   - 高风险操作用完整说明，普通页面只引用短句，避免每个页面重新解释一遍。
