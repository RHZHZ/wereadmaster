# 书籍详情页排版改造方案

## 结论

书籍详情页需要改版，但不推翻现有功能，只收口排版和层级。

当前页面的问题不在数据，而在信息组织方式：简介、元信息、目录、本地整理状态、AI / 相似 / 阅读指南动作分散在多个区域，首屏重心不稳，容易让页面既像信息页，又像工具页，缺少一个明确的主次结构。

这次改造的目标是把详情页收成一个更清楚的“书籍工作台”：

- 先看这是什么书。
- 再看当前读到哪。
- 然后决定继续读、做笔记、复盘还是扩展。
- 目录保持独立，不和简介争首屏。

## 当前问题

1. 首屏层级不稳

- 书名、作者、进度、主按钮之间有主次，但下方信息区又继续抢注意力。
- 简介过长时，会把目录和管理动作挤得太靠下。

2. 信息区过散

- 简介、出版社、ISBN、出版时间、完成时间等内容语义上属于“书籍信息”，但视觉上没有统一收口。
- 目录列表和元信息同时存在时，页面像两个平面并排，缺少明确的阅读路径。

3. 管理动作偏多

- 查看笔记、AI 复盘、找相似、本书阅读指南、候选入口都很重要，但不能同权。
- 当前更适合“主动作 + 次动作”的结构，而不是所有按钮平铺。

4. 纸感方向需要克制

- 详情页可以更像书页，但不应该继续向笔记页那种强纹理方向加重。
- 这里更适合“纸白、细边框、弱阴影、少装饰”，让内容本身成为主角。

## 改造目标

1. 一眼看清四件事

- 书名和作者是谁。
- 当前进度到哪。
- 本机状态是什么。
- 现在能做什么。

2. 结构清晰

- 顶部是书籍 hero。
- 中间是本书管理。
- 底部是简介 / 元信息 / 目录。

3. 目录独立

- 目录列表保持固定高度，内部滚动。
- 不让目录把整页拉长。

4. 保留纸感，但不加重

- 继续使用纸白、浅边框、柔和阴影。
- 不引入更重的纹理或装饰线。

## 建议布局

```text
[返回]
[书籍 Hero]
  [封面 | 分类 / 书名 / 作者 / 标签 / 进度 / 主按钮]

[本书管理]
  [本地状态]
  [查看笔记] [AI 复盘] [找相似] [本书阅读指南]
  [候选入口]

[内容区]
  左侧：简介 + 元信息
  右侧：目录列表（独立滚动）
```

## 视觉原则

### 1. Hero 只负责“识别”

- 封面、书名、作者、分类、进度放在第一屏。
- 主按钮保持 1 到 2 个最强动作，其余动作降级到次级区域。
- 标题最多两行，避免长书名把 hero 撑散。

### 2. 简介只负责“快速理解”

- 简介默认只展示短摘要。
- 过长时折叠或截断，不直接撑满左侧信息区。
- 出版社、ISBN、出版时间等放到元信息区，不和简介抢视觉。

### 3. 目录只负责“继续读”

- 目录应像一个独立浏览列表，而不是信息附录。
- 列表固定高度，内部滚动。
- 当前章节高亮要明确，但不要过度强调每一项。

### 4. 动作要分层

- 第一层：继续阅读 / 在微信读书中打开。
- 第二层：查看笔记、AI 复盘、找相似、本书阅读指南。
- 第三层：候选入口、状态清除等低频动作。

## 组件落点

建议保持现有组件拆分，只调整职责边界和布局密度：

- `BookHeader`
  - 继续承担 hero 区。
  - 精简标题换行和进度展示。
  - 让主按钮更聚焦。

- `BookActionPanel`
  - 明确“本书管理”语义。
  - 把高频动作和低频动作分层。
  - 不再让所有动作同权平铺。

- `book-info-panel`
  - 简介改为更短的主摘要。
  - 元信息区保持规整对齐。

- `ChapterList`
  - 独立成可滚动区域。
  - 保持目录可扫读，不挤压左侧信息区。

## 响应式建议

### 宽屏

- 保持 hero + 管理区 + 内容区的三段式。
- 内容区可继续采用左右两栏，但右栏只给目录。
- 左侧信息区宽度要足够容纳简介，不要过窄。

### 窄屏

- hero、管理区、简介、目录改为单列堆叠。
- 目录放到后面，避免首屏太重。
- 主按钮和次按钮改成可换行的紧凑布局。

## 明确不做

- 不把书籍详情页改成笔记页。
- 不引入新的图表或复杂交互。
- 不新增后端字段。
- 不把目录改造成多列导航。
- 不把所有动作都做成显眼卡片。

## 实施顺序

1. 先收口 `BookHeader` 的主视觉和标题密度。
2. 再调整 `BookActionPanel` 的动作分层。
3. 然后让简介 / 元信息区更克制。
4. 最后把目录做成独立滚动区并验证窄屏表现。

## 验收标准

- 首屏能同时看清书名、作者、进度和主动作。
- 简介不会把目录挤出首屏太多。
- 目录列表独立滚动，页面不会被章节撑得过长。
- 本书管理动作有明确主次，不会视觉打架。
- 纸感保留，但没有过重纹理、阴影或装饰线。
- 桌面端和窄屏都不发生文字溢出、按钮重叠或布局跳动。

## 风险

- 书名过长时，hero 可能再次被拉高，需要限制标题行数。
- 简介过长时，如果不做折叠，还是会压缩目录区。
- 如果动作区继续平铺，详情页会重新回到“工具面板”感。

## 备注

这份改造只针对排版和层级，不改变书籍详情页的数据能力与操作能力。保留当前功能，先把页面讲清楚，再谈更细的视觉优化。
