# Product Audit

## 结论

当前产品已经具备“本地优先微信读书管理工具”的核心雏形：微信书架、候选书架、详情、笔记、统计、AI 复盘、发现和设置都能形成闭环。下一阶段不应该继续横向堆功能，而应该把工具从“能看数据”推进到“能指导下一步阅读管理”。

优先级判断：

- P0：收口现有页面的信息架构、空态、错误态和入口重复。
- P1：补齐阅读管理闭环，让用户知道下一步读什么、复盘什么、整理什么。
- P2：增强 AI，但只做基于本地资料的结构化辅助，不做泛聊天。

## 产品定位

目标用户是已经长期使用微信读书、希望把阅读资产沉淀到本地的人。产品应该强调：

- 本地优先：缓存、凭据、AI 输出都在本机。
- 可行动：每个页面都应该回答“我现在可以做什么”。
- 可回溯：统计和 AI 结论必须能解释来源。
- 少干扰：不自动上传、不自动生成、不制造伪智能结论。

微信读书 skill 的最终输出必须落到三类价值之一：

- 行动：让用户知道下一步读什么、复盘什么、整理什么，且能直接进入对应页面或状态。
- 决策：帮助用户在候选书、复盘对象、阅读顺序之间做取舍，而不是只展示更多信息。
- 交付物：生成可保存、可迁移、可分享或可继续写作的 Markdown、索引、报告、卡片或行动清单。

统计、标签、趋势和 AI 文案只能作为依据，不能作为最终价值。后续新增能力如果不能触发行动、节约时间或形成交付物，应默认不做。

## 微信读书 skill 应用场景

这些应用场景是工作流，不是页面清单：

- 下一步阅读指挥官：聚合继续读、待复盘、本地候选、下周期建议，只给少量明确动作。
- 单本书阅读处方：围绕当前书给出阅读任务、复盘点、建议输出物和验收标准。
- 笔记复盘助手：把划线和想法转成关键观点、行动项、复盘问题和可处理状态。
- 阅读资产打包器：批量导出笔记、已生成复盘、索引和报告，节约迁移、备份和写作整理时间。
- 选书决策助手：基于当前书、候选书、已读主题和本地状态解释为什么下一本读它，而不是泛推荐更多书。
- 输出素材生成器：把阅读资产整理成复盘卡片、写作提纲、行动清单或分享素材，但不伪造未提供的书籍内容。

当前实现状态：

- 下一步阅读指挥官：已具备总览聚合，但还需要继续把动作数量收敛为少量决策输出。
- 单本书阅读处方：已通过本书阅读指南、推进任务、复盘点、下一步行动和导出形成闭环。
- 笔记复盘助手：已通过单本 AI 复盘、行动项处理状态、复盘问题轻量回答状态和导出形成闭环。
- 阅读资产打包器：已通过批量导出、复盘导出、失败报告、重试、备份恢复和导出位置形成基础闭环。
- 选书决策助手：首版已实现，只围绕候选书架里的本地候选做取舍，不做泛推荐。
- 输出素材生成器：已有分享图片、Markdown、复盘复制和行动清单导出基础，后续再评估复盘卡片包、写作提纲和周报素材。

不建议把产品扩展成：

- 通用书籍搜索网站。
- 泛用 AI 聊天工具。
- 大而全的知识库系统。
- 需要用户长期维护复杂字段的任务管理器。

## 页面审计

### 总览

定位：进入应用后的阅读控制台，提供状态感和最快行动入口。

当前状态：

- 已有书架、笔记、复盘、发现快捷入口。
- 凭据状态已降级为细状态条，保留设置入口但不再作为大卡片抢占视觉权重。
- 已加入最近打开内容，能从本地缓存进入书籍详情。
- 已加入“今日可做”模块，优先聚合继续读、待复盘、本地候选和复盘入口。
- 已加入“本地队列”模块，把继续读、待复盘、本地候选三类下一步动作集中到总览；仅读取书架缓存、已加载笔记概览和本地 `reading_item_states`，不新增后台同步。
- 已加入近期阅读画像和推荐书籍摘要：画像只读取本地月度统计缓存，推荐书籍优先使用微信读书推荐接口，接口失败或无结果时降级为发现页保存的本地候选封面；不自动调用 AI。
- 右侧指标和底部最近内容已经开始向可行动方向收敛。

问题：

- 首屏对新用户和老用户的差异化还不够明显。

建议：

- 继续强化“今日可做”的优先级排序，例如接入统计异常和下周期建议，但仍只基于本地缓存或用户已触发生成的结果。
- 保持最近内容只显示可行动条目，避免有声书或文章点击后无结果。

### 书架

定位：个人阅读资产列表，用于筛选、选择和进入详情。

当前状态：

- 支持同步、类型筛选、父分类筛选、搜索和电子书详情入口。
- 侧边栏书架应拆成 `微信书架` 和 `候选书架` 两个二级入口：微信书架承接同步与浏览；候选书架承接本地候选管理和“推荐下一本”入口。
- 电子书点击卡片直接进入详情，不再经过“选中条目”中间层。
- 有声书和文章收藏通过卡片右上角三点菜单提供“复制标题”和“去发现页搜索”动作，不再撑高卡片。
- 右侧 selection-panel 已移除，书架回到纯网格浏览，减少宽屏下的无效空白。
- 分类区默认只展示高频父分类，低频分类通过“展开更多”查看，避免首屏被分类 chip 占满。

问题：

- 书架是资产列表，微信读书已有的在读、已读、置顶状态不应在本地重复管理；本地只补充微信读书没有覆盖的候选、复盘、整理意图。
- 候选书架不是微信读书远端书架，必须明确“只保存在本机，不写回微信读书”。
- 有声书和文章收藏还不能进入详情页，当前只提供轻量替代动作。

建议：

- 短期保持非电子书轻量动作，不伪装成完整详情页。
- 中期：增加本地标记层，不修改微信读书数据；发现页结果只负责保存本地候选，候选书架负责管理候选和发起选书决策，微信书架内书籍只使用“待复盘 / 已整理”。

### 书籍详情

定位：单本书的阅读进度、元数据、章节和外部打开入口。

当前状态：

- 能展示基础信息、进度、章节、微信读书打开入口和相似书入口。
- “AI 阅读指南 / 跨书路线图”已经完成预研，适合放在详情页的本书管理区，但应保持手动触发：默认基于当前书、已生成复盘、统计标签和本地状态生成本书阅读指南；只有用户显式选择候选书后，才扩展为跨书路线图。不在进入详情页时自动调用。
- 章节数据适合做目录展示、章节分组、回跳和引用来源，但不稳定到足以支撑实时章节任务派发。

问题：

- 详情页更像“接口数据展示”，还没有成为单本书的管理页。
- 单本笔记、AI 复盘、相似推荐之间的入口可以更紧密。
- 如果把完整 AI 路线内容直接铺在详情页，会让详情页从管理页变成内容页，和笔记/复盘中心争夺主职责。
- 如果把阅读追踪理解为“实时知道用户读到哪一章”，会过度依赖微信读书接口和客户端状态；当前更可靠的方式是按阅读进度阶段触发下一步建议。

建议：

- 增加单本书的本书管理区：查看笔记、生成复盘、找相似、标记待复盘或已整理；不要在书架内书籍上重复标记待读/在读。
- 保持详情页不承载长 AI 内容，只做入口；AI 阅读指南作为“本书阅读指南”按钮进入独立结果页。
- 首版 AI 阅读指南应先进入独立结果页，候选书默认不纳入；用户选择候选书并确认输入范围后，才切换为跨书路线图并允许生成和导出。
- 阅读追踪第一版采用“进度阶段驱动”：`0-15% 起步`、`15-40% 建立主线`、`40-70% 深入推进`、`70-95% 收束整理`、`100% 完成归档`。阶段只用于提示刷新指南、整理笔记或生成复盘，不自动调用 AI。
- 章节数据只作为阶段建议的增强依据；缺章节或目录未缓存时，回退到进度百分比、最近笔记、本地状态和已有复盘摘要。

### 笔记中心

定位：有笔记书籍的索引和进入单本笔记的入口。

当前状态：

- 能同步笔记、搜索书籍、展示笔记数量并进入单本笔记。

问题：

- 缺少“最近有新增笔记”“最值得复盘的书”这类管理视角。
- 用户打开笔记中心后，主要仍是在找书，而不是被引导整理。

建议：

- 增加基于本地缓存的“待复盘笔记”排序：按新增笔记数、最近更新时间、未生成 AI 复盘优先。
- 搜索保留，但不要让搜索成为唯一组织方式。

### 单本笔记

定位：阅读原始摘录、想法和导出入口。

当前状态：

- 支持读取、导出 Markdown、进入 AI 总结。
- 支持章节视图和卡片视图切换，默认仍保留按章节阅读的上下文。
- 卡片视图支持全部/划线/想法筛选、按章节/最新排序、随机一组、单条笔记分享图片和当前组组合分享图片。
- 想法/点评已展示其依附原文、位置和章节回跳信息，避免脱离原文单独出现。
- Markdown 导出已吸收官方导出的可取部分：增加 front matter、封面/进度元数据、人类可读时间、划线回跳链接和稳定块引用 ID；仍保留通用 Markdown 为默认，不强绑定 Obsidian。
- 明确书签内容不可导出的限制。

问题：

- 摘录和想法仍缺少本地整理动作。
- AI 总结入口存在，但用户不一定知道何时值得生成。
- 如果直接改成瀑布流，会打散章节和时间顺序，不适合作为默认阅读型视图。
- Markdown 导出已适合归档和二次索引；有依附原文的想法已经能和原文成组展示，不能可靠定位的独立想法仍按章节单独展示。

建议：

- 默认保留章节分组或时间顺序视图，保证笔记上下文可回溯。
- 导出格式短期保持通用 Markdown；如后续增加 Obsidian 增强模式，再引入 callout，避免默认格式过度绑定单一工具。
- 下一步补“适合复盘 / 数据不足”判断，并在进入 AI 总结前提示输入规模：划线数、想法数、章节覆盖。
- 如要继续增强单本笔记，优先做本地整理状态或空态引导，不再增加新的浏览模式。

### 单本 AI 复盘

定位：基于单本笔记的结构化复盘页。

当前状态：

- 支持缓存读取、重新生成、导出。
- 结构包括概览、标签、观点、关注点、行动项、摘录、问题和来源统计。
- 支持关键观点、我的关注点、行动与复盘、复盘问题和完整复盘复制。
- 行动项已支持本地四态反馈、反馈记录和轻量行动清单导出；复盘问题已支持轻量回答状态和回答记录，并与行动项写入统一复盘反馈状态。
- 页面内展示 AI 复盘数据边界，说明缓存、手动生成和笔记发送时机。
- AI 页面与 Markdown 导出已补充结构化约束元信息，可区分本次请求实际使用 `JSON Schema` 还是兼容回退的 `JSON Object`。

问题：

- 内容已经比较完整，但容易偏“读报告”而不是“下一步执行”。
- 复盘问题仍主要是阅读提示，但已补轻量回答状态；行动项已进入最小反馈闭环。

建议：

- 后续只评估复盘问题是否需要更正式的回答归档；行动项暂不扩展成任务系统。
- 暂不做个人笔记问答，避免引入索引和防幻觉复杂度。

### 统计

定位：结构化阅读数据仪表盘。

当前状态：

- 支持周期切换、总时长、阅读天数、日均、环比、趋势、最长内容、作者偏好和分类偏好。
- AI 大卡已移除，只保留轻量复盘入口。
- 长读书目、作者偏好、分类偏好都改为卡片内局部滚动；作者偏好和分类偏好使用全宽卡片，避免两列布局留下大块空白。
- 已增加本地统计解读，基于当前缓存解释投入最多分类、最长内容占比、节奏集中度和周期变化，不调用 AI。

问题：

- 总计模式下数据很大，页面需要更强调长期资产而非周/月节奏。

建议：

- 继续强化总计模式下的长期资产解释，例如近年高峰和长期投入主题。
- 继续保持统计页不承载长 AI 文案。

### 复盘中心

定位：AI 产出的集中入口，包括单本复盘列表、阅读指南库和阅读报告。

当前状态：

- 侧边栏下有“书籍复盘”和“阅读报告”二级入口。
- 书籍复盘页同时展示已生成复盘和有笔记但尚未复盘的候选书。
- 候选书进入单本复盘页时只使用本地笔记统计，不自动读取或发送笔记正文。
- 阅读报告页已经承载趋势时间线、习惯画像、偏好地图、重点内容和行动建议。
- AI 阅读指南 / 跨书路线图已有详情页手动入口、独立结果页、缓存、导出和页面内完整展示，但还缺少按书聚合的管理入口。

问题：

- “复盘中心”的概念还可以更强，现在更像两个页面拼在一起。
- 复盘中心已具备“已生成 / 建议生成”的基本管理形态，后续重点是和总览、笔记中心形成更强闭环。
- 阅读指南和跨书路线目前仍是从书籍详情进入的单次结果页，命中缓存后可以展示，但用户不能从复盘中心按书找回、比较或管理这些 AI 产出。
- 如果直接增加“生成历史”列表，数据多后会变成按时间堆叠的 AI 输出垃圾堆，不能服务行动和节约时间。

建议：

- 继续保持生成必须由用户点击触发，不做后台自动生成。
- 后续可把待复盘候选轻量聚合到总览，但不要把复盘中心变成任务系统。
- 在 `复盘` 子菜单中增加 `阅读指南`，进入按书聚合的阅读指南库；主列表展示书籍资产卡片，不展示所有历史版本。
- 阅读指南库第一版只做聚合入口：书名、作者、当前进度、计算阅读阶段、本地追踪状态、本书指南、跨书路线数量、书籍复盘状态、刷新建议原因和最近更新时间。
- 点击书籍资产卡进入书籍资产详情，页面内用 `阅读指南 / 跨书路线 / 书籍复盘` 标签组织当前有效内容。
- 历史版本、跨书路线接续、长文本行动反馈和 prompt v3 后置评估；第一版只做 `reading-route-v2.1` 小版本提示词收紧，不把这些能力一次性塞进页面。
- 长内容继续由独立页面或详情页承载；弹窗只用于短确认、少量候选选择、发送长文本确认和归档确认。

### 发现

定位：从个人阅读资产出发找下一本书。

当前状态：

- 支持搜索、范围切换、推荐、书架种子、主题 chip、最近搜索。
- 搜索后页面进入轻量搜索态：Hero 压缩，搜索结果前置，书架种子和主题/最近搜索降级为辅助区。
- 搜索态已进一步收口为结果优先：搜索结果占满主列，右侧推荐不再常驻，书架种子、本地候选、主题和最近搜索压缩为轻量辅助条。
- 搜索结果和相似探索保留信息卡形态，便于比较标题、作者、评分和操作；封面比例已固定，不再因卡片内容高度被拉成长条。
- 默认态个性化推荐已从右侧纵向栏改为封面主导的横向推荐带，卡片主体打开详情，次级动作收进右上角三点菜单，降低推荐区高度并减少页面右下空白。
- 推荐卡片评分已兼容微信读书推荐接口的 `newRating`、`ratingPercent`、`rating` 以及 `bookInfo` 包裹结构；默认推荐区不强行展示评分，完整评分下沉到详情页，避免推荐接口缺字段时出现整排“暂无评分”。
- 默认态底部辅助区已对齐搜索后的轻量辅助条布局：本地候选、书架种子、主题和最近搜索并列展示。
- 发现页主内容容器已取消固定 `max-width`，宽屏下书架种子、推荐封面带和辅助入口会跟随工作区自适应展开。
- 相似推荐已从右侧常驻面板改为独立“相似探索”模式：从书架种子、搜索结果或详情页进入，顶部显示种子书和返回发现入口。

- 不读取全量书架，不同步远端笔记，不发送原始划线/想法/章节正文，不写回微信读书。
- 选书决策助手不替代阅读路线图：前者回答“下一本先读谁”，后者回答“当前书和候选书如何排顺序推进”。

- [x] 数据目录迁移会同时处理 SQLite 主文件、WAL 和 SHM 文件。
- [x] 目标目录不存在、不可写、被删除或位于不可访问设备时，应用能给出可恢复错误。
- [x] 本地数据库位置变更需要用户确认，并说明 API Key 不随数据库迁移。
- [x] 导出保存位置变更不触发数据库迁移，设置页明确只影响后续导出文件。
- [x] 诊断导出能显示当前数据目录、数据库路径和最近一次迁移/恢复错误。

### 阶段 6：批量导出向导

目标：提供真正可用的一键导出能力，同时保持本地优先和用户确认边界。批量导出可以读取已缓存内容，也可以在用户明确确认后同步缺失笔记；不能静默批量请求远端，不能批量生成 AI 复盘。

信息架构决策：

- 笔记页批量导出以划线/想法 Markdown 为主，不继续叠加复盘专属筛选、搜索和批量操作。
- 笔记页可以兼容“随书籍附带导出已有复盘缓存”，但这不是批量导出向导的主任务。
- 书籍复盘的独立批量导出归属“书籍复盘”页，由步骤弹窗承接已生成复盘搜索、显式勾选、设置确认、导出结果和复盘索引。
- 批量生成复盘不属于导出流程；如后续需要，必须另做显式确认和生成队列，不和“导出书籍复盘”混在一起。

当前交互约定：

- 批量导出弹窗以书籍列表为主工作区；预检统计、策略选择和导出设置必须保持紧凑，不能挤压列表可视高度。
- 弹窗外层不滚动，书籍列表区独立滚动，底部“重新预检 / 开始导出”操作区固定可见。
- 桌面宽度下，选择“只导出选中的书”时，设置工具栏必须保持一行：`排除无划线/想法的书`、搜索框、`已选择 N 本` 从左到右排列。
- 桌面宽度下，选择“先同步缺失笔记再导出”时，设置工具栏必须保持一行：同步并发选择、`排除无划线/想法的书` 从左到右排列。
- `排除无划线/想法的书` 默认开启；详细说明不占据主视觉高度，可通过控件提示承载。只有书签的书默认不进入一键导出队列，取消过滤后显示为无内容项。
- 窄屏下工具栏可以纵向堆叠，但不能产生水平溢出，且书籍列表仍应优先保留可读空间。
- 书籍复盘导出弹窗不照搬笔记批量导出同步策略，只保留“选择复盘 -> 确认设置 -> 导出结果”的轻流程；默认不全选，用户必须显式选择。
- 书籍复盘导出设置只控制本地 Markdown 内容：行动反馈、复盘问题反馈和代表性摘录默认全开；关闭后不影响本地缓存、不触发重新生成，也不请求微信读书远端。

- [x] 增加批量导出预检：扫描本地笔记概览、单本笔记缓存和已生成复盘缓存，统计可导出、需同步、无内容、失败风险。
- [x] 增加导出策略选择：仅导出本地已缓存内容、先同步缺失笔记再导出、只导出用户选中的书籍。
- [x] 增加缺失笔记同步队列：使用 2-3 并发上限，每本书独立任务，支持取消和失败记录。
- [x] 增加缺失笔记同步失败后的单本重试入口：导出报告中的失败书籍可单独重试，只重跑该书同步和导出。
- [x] 增加批量笔记 Markdown 导出：每本书一个 Markdown，并生成 `index.md` 汇总目录。
- [x] 笔记批量导出兼容附带已生成书籍复盘：只导出本地已有 AI 复盘缓存，不自动生成缺失复盘。
- [x] 增加导出报告：生成 `export-report.md`，记录成功、跳过、同步失败、无可导出内容、未生成复盘等原因。
- [x] 增加默认过滤选项：默认排除没有划线/想法的书；只有书签的书不进入一键导出队列，用户取消过滤后才显示为无内容项。已有 AI 复盘缓存的书即使没有划线/想法也保留。

建议开发顺序：

1. 先做预检和只导出本地缓存，建立导出报告格式。
2. 再做用户确认后的缺失笔记同步队列，默认并发 2，最多 3。
3. 已完成：书籍复盘页提供独立导出入口，通过步骤弹窗导出用户显式选择的已生成复盘缓存并生成索引。
4. 已完成：同步失败后在报告中提供单本重试入口，重试范围限定为失败书籍。
5. 最后再评估是否需要 zip 打包；首版可以先输出目录。

验收标准：

- [x] 一键导出前必须展示预检结果和导出策略，不能静默批量请求微信读书接口。
- [x] 选择“先同步缺失笔记再导出”后，队列并发数受控，单本失败不阻断整体导出。
- [x] 用户取消同步时，已完成的缓存可以继续导出，未完成项写入导出报告。
- [x] 同步完成但仍没有划线/想法时，结果记为跳过，不按失败处理。
- [x] 导出结果包含 `index.md` 和 `export-report.md`。
- [x] 笔记批量导出附带的书籍复盘只使用已生成缓存，不触发 AI 生成。
- [x] 书籍复盘页补独立导出入口和步骤弹窗，替代在笔记批量导出向导中继续堆叠复盘专属设置。
- [x] 书籍复盘导出确认步骤支持真实导出开关：包含行动反馈、包含复盘问题反馈、包含代表性摘录，默认全开。
- [x] 导出内容不包含 API Key、数据库路径、原始接口响应或敏感日志。
- [x] E2E 覆盖仅本地缓存导出、确认同步后导出、取消流程、默认过滤无划线/想法书籍和选中策略搜索。
- [x] E2E 覆盖批量导出书籍列表独立滚动、操作区固定可见、桌面宽度下书籍列表最小可视高度和选中策略工具栏同排显示。
- [x] E2E 补充真实失败报告场景，并覆盖失败后的单本重试。

## 不做清单

短期不做：

- 自动后台生成 AI 总结。
- 批量自动生成 AI 复盘；批量导出只能导出已生成复盘。
- 个人笔记自由问答。
- 向量索引和语义搜索。
- 改写微信读书远端数据。
- 复杂目标管理、打卡系统或社交功能。
- 直接手填数据库路径；数据库存储位置自定义必须等备份、恢复、迁移和路径失效处理成熟后再开放。导出保存位置不属于数据库迁移，优先使用系统目录选择器，手动输入只作为兜底并校验可写目录。
- 完整首次初始化向导；短期只保留轻量 checklist 或空态引导。
- 自由主题色、多语言、自定义字体文件和复杂个性化；近期只做预设主题、字号、信息密度和默认入口偏好。

这些功能不是没有价值，而是会显著增加数据边界、引用策略、防幻觉和状态同步复杂度，应该等现有阅读管理闭环稳定后再评估。
