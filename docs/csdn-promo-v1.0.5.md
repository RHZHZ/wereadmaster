你有没有遇到过这种情况：

- 微信读书里读过很多书，但最后只记得“我好像读过”。
- 划线和想法越积越多，真正整理时却要一本本翻。
- 想复盘一本书，发现摘录、进度、统计和输出都散着。
- 月底想看看自己读了什么，却没有一份能保存、能分享的阅读报告。

我自己就是被这些问题反复卡住，所以做了一个桌面端工具：**WeReadMaster，微信读书个人阅读管理**。

它不是另一个阅读 App，而是一个本地优先的阅读工作台。它做的事情很具体：把微信读书里的书架、笔记、统计和 AI 复盘，整理成你自己电脑里的长期知识资产。

你可以先把它理解成三件事：

- **把微信读书数据收回来**：书架、笔记、阅读统计优先落到本地。
- **把笔记变成可复盘内容**：单本书可以生成结构化复盘、行动项和复盘问题。
- **把统计变成分享图**：月报、卡片组和横版报告可以保存下来，方便复盘和分享。

## 数据从哪里来：走微信读书官方 Skills，不托管账号

这点我会放在前面说清楚，因为它直接关系到用户是否放心使用。

WeReadMaster 读取微信读书数据时，走的是 **微信读书官方 Skills HTTP 接口**。用户需要在微信读书官方页面扫码获取自己的 API Key，然后在本地应用里填写。

它不是这些方式：

- 不是让你输入微信账号密码。
- 不是抓 cookie。
- 不是托管你的微信读书账号。
- 不是绕过付费内容。
- 不是读取别人的数据。

简单说，它只帮助你把自己账号里已经合法拥有的书架、笔记和统计同步到本地，再做整理、复盘和导出。

## 5 个用户能直接感受到的亮点

### 亮点一：打开以后，不用再到处找入口

打开应用以后，你不用在多个入口之间来回切。

书架、笔记、统计、复盘、发现和设置都在一个桌面端里。同步以后，即使不联网，也能查看本地已有缓存。

![总览工作台](https://raw.githubusercontent.com/RHZHZ/wereadmaster/main/src/assets/hero-reading-dashboard.jpg)

我希望它更像一个每天会打开的阅读工作台，而不是一块好看的数据大屏。你进来以后能快速知道：最近在读什么、哪些笔记值得整理、下一步应该继续读，还是先复盘。

### 亮点二：划线不再只是收藏，可以变成复盘材料

很多人划线以后就没有下一步了。

WeReadMaster 会把单本书的划线和想法整理成更适合复盘的结构，包括：

- 这本书的核心观点。
- 哪些摘录最值得回看。
- 可以落地的行动项。
- 适合继续思考的复盘问题。
- 后续是否需要重新生成新版本。

也就是说，AI 在这里不是陪聊，而是帮你把阅读材料加工成更容易保存和输出的内容。你可以把它当成一个“读后整理助手”：不替你读书，但帮你把已经读过的东西重新组织起来。

### 亮点三：读到一半卡住时，它能给出下一步阅读建议

有些书不是读完才需要复盘，读到一半也会卡住。

这个工具可以根据当前书、阅读进度、已有笔记和统计信号，生成一份阅读指南。它会尽量回答这些问题：

- 这本书现在处在什么阅读阶段？
- 接下来应该重点读哪一部分？
- 什么时候适合停下来整理？
- 读完以后可以产出什么？
- 如果要延展同一主题，下一本可以怎么选？

对我来说，这比单纯问 AI “帮我总结这本书”更有用。它会结合当前阅读状态，而不是只生成一篇漂亮但用完即走的摘要。

### 亮点四：本地优先，阅读记录先属于你自己

这个项目非常强调本地优先。

书架缓存、笔记、阅读状态、AI 输出、导出文件，都尽量放在用户自己的电脑里。AI Provider 也由用户自己配置，可以使用 OpenAI 兼容服务或本地模型服务。

更重要的是：

- 不托管你的微信读书账号。
- 不保存明文 API Key 到前端日志。
- 不在后台自动上传笔记给 AI。
- 不破解、不爬取他人数据、不绕过付费内容。
- AI 调用需要用户手动触发。

![本地优先工作台界面](https://raw.githubusercontent.com/RHZHZ/wereadmaster/main/src/assets/hero-reading-dashboard.jpg)

如果你在意数据边界，希望阅读记录先属于自己，这一点会很重要。尤其是笔记和复盘这种带有个人判断的内容，我更倾向于先放在本机，再由用户自己决定是否导出、分享或同步到别处。

### 亮点五：不只微信读书，本地图书也能一起管理

除了微信读书，它也支持导入本地图书，目前包括 EPUB、TXT 和 Markdown。

本地图书和微信书架里的同名书不会被强行合并。它们会作为两个来源版本保存，阅读进度、划线、想法和 AI 记录彼此独立。

这个设计看起来谨慎，但实际很必要。因为不同来源的书排版、章节、位置都可能不一样，强行合并很容易让笔记错位。阅读资产最怕的不是少一点自动化，而是数据变得不可解释。

## 适合哪些用户

我觉得它特别适合这几类人：

- 长期使用微信读书，书架和笔记已经积累很多。
- 希望把阅读记录导出到 Markdown、Obsidian、Notion、博客或自己的知识库。
- 想定期做阅读复盘，但不想每次从零整理。
- 希望 AI 帮忙总结和规划，但不想把所有内容交给中转平台。
- 想要一个更稳定、更私密的桌面端阅读管理入口。

如果你只是偶尔看书，可能微信读书原 App 已经足够。但如果你真的想把阅读变成一个长期积累系统，这个工具会更有价值。

## 怎么开始使用

当前正式版本主要面向 Windows x64。

1. 打开 Releases 页面：  
   https://github.com/RHZHZ/wereadmaster/releases

2. 下载最新 Windows 安装包。

3. 安装并启动应用。

4. 在设置页配置微信读书 API Key。

5. 如果要使用 AI 复盘，在 AI 设置里填写自己的 Provider 和 Key。

6. 同步书架和笔记以后，就可以查看统计、生成报告、整理复盘、导出 Markdown。

微信读书数据读取走官方 Skills HTTP 接口，API Key 需要用户自己在官方页面扫码获取。AI Key 也由用户自己配置，本项目不提供中转服务。

## 最后

我做这个工具的初衷很简单：读过的书，不应该只剩一个进度条；划过的线，也不应该永远散在 App 里。

阅读本来就是慢慢积累的事。一个好的工具，不一定要替你读书，但应该帮你把读过的东西留下来、整理好、在需要的时候重新变成可用的知识。

如果你也在用微信读书，也希望把书架、笔记、统计和复盘沉淀到自己电脑里，可以试试 WeReadMaster。
