# 书籍详情页视觉密度改造方案

## 结论

当前书籍详情页的首屏视觉尺度偏大，需要收紧。

问题不在功能，而在视觉权重：顶部 hero 更像营销展示页，标题、封面、进度数字和按钮同时放大，导致页面信息密度下降。对于书籍详情页这类工具型界面，用户更需要快速扫读书名、作者、进度和可执行动作，而不是被一个超大首屏占满注意力。

这次改造不改变数据结构、不新增功能，只调整尺寸体系、间距和层级，让页面从“展示型 hero”回到“书籍工作台”。

## 当前问题

1. 标题过大

- 当前 `BookHeader` 标题使用 `clamp(30px, 4vw, 52px)`。
- 在宽屏下标题接近 52px，长书名会形成过强视觉压迫。
- 标题已经超过工具型详情页需要的识别权重。

2. 封面偏大

- 当前 `.book-cover-large` 宽度为 `180px`，最小高度为 `260px`。
- 截图中封面和标题共同撑高首屏，使卡片更像宣传展示位。
- 封面应负责识别书籍，而不是主导页面视觉。

3. 首屏留白偏松

- `.book-header` 使用 `24px` padding 和 `24px` gap。
- `.book-header-copy` 使用 `14px` gap。
- 元信息、进度和按钮之间的节奏偏散，降低扫读效率。

4. 进度数字权重偏高

- `.progress-block strong` 当前为 `34px`，在截图中已经接近标题级别。
- 对于 `100%` 这种明确状态，不需要用大字号重复强调。
- 进度条本身已经提供了状态确认。

5. 标签和按钮偏厚

- `.book-meta-row span` 使用 `8px 11px` padding。
- 主按钮和次按钮高度较高，视觉上进一步扩大首屏。
- 书籍详情页需要保留可点击面积，但不需要营销页式的大按钮。

## 改造目标

1. 提升信息密度

- 首屏能同时看到书名、作者、评分、阅读时长、进度和主动作。
- 书名不因字号过大挤压后续信息。
- 卡片看起来更像详情工作台，而不是封面展示页。

2. 降低 hero 权重

- hero 只负责识别书籍和承载最关键动作。
- 视觉重心从“巨大标题”回到“书籍状态 + 操作路径”。

3. 保持可读和可点

- 字号收紧但不牺牲阅读性。
- 按钮收紧但仍满足桌面端点击舒适度。
- 移动端仍保持单列和足够触控高度。

4. 不破坏现有结构

- 优先调整 `src/styles.css`。
- 只有在 CSS 无法表达层级时，才小幅调整 `BookHeader` 结构。
- 不改接口、不改数据、不改业务行为。

## 建议尺寸

### 桌面端

| 元素 | 当前倾向 | 建议目标 |
| --- | --- | --- |
| hero 标题 | 最大约 52px | `clamp(30px, 3vw, 40px)` |
| 标题行高 | 1.12 | `1.15` 到 `1.2` |
| 封面宽度 | 180px | 156px 到 168px |
| 封面高度 | min-height 260px | 220px 到 240px |
| hero padding | 24px | 20px 到 22px |
| hero gap | 24px | 18px 到 20px |
| 内容纵向 gap | 14px | 10px 到 12px |
| 元信息 pill padding | 8px 11px | 6px 10px |
| 进度数字 | 34px | 28px 到 30px |
| 进度条高度 | 14px | 8px 到 10px |
| 按钮图标 | 18px | 16px 到 18px |

### 窄屏

- hero 改为单列后，封面宽度控制在 `min(148px, 45vw)`。
- 标题最大不超过 `32px`。
- 元信息 pill 可以换行，但不要强制变成纵向列表。
- 主按钮可占满一行，次按钮保持正常内容宽度。

## 组件落点

### `BookHeader`

文件：`src/components/BookHeader.tsx`

建议保持当前组件职责，不拆新组件。

- 保留封面、分类、书名、作者、元信息、进度和主动作。
- 不新增描述文案。
- 不把 `找相似书` 升级成和主按钮同等强度的视觉按钮。
- 长标题需要限制最大行数，避免再次撑高 hero。

### 书籍详情页样式

文件：`src/styles.css`

优先调整以下选择器：

- `.book-header`
- `.book-cover-large`
- `.book-header-copy`
- `.book-header h3`
- `.book-meta-row`
- `.book-meta-row span`
- `.progress-block`
- `.progress-block strong`
- `.progress-block meter`
- `.book-actions`

建议新增或调整标题截断规则：

```css
.book-header h3 {
  display: -webkit-box;
  overflow: hidden;
  -webkit-line-clamp: 2;
  -webkit-box-orient: vertical;
}
```

如果需要兼容非 WebKit 环境，可接受退化为普通换行，核心要求是避免桌面端长书名无限撑高。

## 建议视觉层级

```text
一级：书名
二级：作者、进度状态、主按钮
三级：评分、阅读时长、读完时间
四级：找相似、当前章节提示
```

具体表现：

- 书名仍是最大文本，但不超过 40px。
- `100%` 不再接近标题大小，改为状态数字。
- 元信息 pill 只做辅助识别，不做醒目标签墙。
- 主按钮保持明确，次按钮降低视觉重量。

## 实施顺序

1. 收紧 `BookHeader` 标题字号和行高。
2. 下调封面尺寸，并同步调整 grid 列宽。
3. 收紧 hero padding、gap 和内容纵向间距。
4. 下调进度数字和进度条高度。
5. 收紧元信息 pill 和按钮尺寸。
6. 补充长标题两行限制。
7. 检查窄屏布局，避免按钮、标题、标签重叠。

## 验收标准

- 宽屏首屏不再呈现营销页式超大标题。
- 书籍 hero 高度明显下降，但书名和封面仍可快速识别。
- `100%` 进度不会抢过书名视觉权重。
- 评分、阅读时长、完成时间可扫读，但不形成厚重标签墙。
- 主按钮明确，次按钮弱一级。
- 长书名最多两行，不把 hero 撑到异常高度。
- 窄屏下封面、标题、按钮不重叠，文字不溢出容器。
- 改造后不影响打开微信读书、找相似书、章节跳转等现有行为。

## 不做范围

- 不重构书籍详情页数据流。
- 不新增后端字段。
- 不新增书籍操作。
- 不把页面改成全新的视觉主题。
- 不引入复杂响应式断点体系。
- 不处理简介、目录和公开评论的深层结构问题；这些属于已有布局改造方案的范围。

## 风险和注意事项

- 标题两行截断可能隐藏部分长书名，需要依赖 `title` 属性或后续细节页补充完整展示。
- 封面缩小后，低清封面会更不明显，但整体页面会更稳。
- 如果只改标题不改间距，页面仍会显得松散；尺寸和间距必须一起收紧。
- 如果按钮仍全部使用高权重样式，首屏会继续偏重，需要同步区分主次按钮。

## 推荐落地标准

第一版建议采用最小改动：

- 只改 `src/styles.css`。
- 不改 `BookHeader` JSX 结构。
- 不新增设计 token。
- 使用现有颜色、边框和纸感背景。

这样符合 KISS 和 YAGNI：先解决明确的尺度问题，再根据实际截图决定是否继续做结构级调整。
