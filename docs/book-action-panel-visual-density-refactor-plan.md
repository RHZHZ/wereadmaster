# 本书管理区视觉密度改造方案

## 结论

当前“本书管理”区域的设计方向是合理的，但视觉密度仍偏低。

它已经具备清晰的管理语义：展示本机整理状态、给出下一步建议、提供笔记 / 复盘 / 相似书 / 阅读指南等动作。但问题在于低频信息占用了过大的展示空间，状态卡、状态选项和候选入口都偏“卡片化”，导致页面看起来比实际功能复杂。

这次改造不改变业务能力，只收紧本书管理区的层级和尺寸，让它更像一个高效的书籍工作台。

## 当前问题

1. 顶部状态卡偏空

- `book-asset-status-card` 占据整行，并包含状态标签、标题、说明、两个元信息块和下一步说明。
- 当前信息是有价值的，但展示方式过厚，容易形成大块空白。
- 对于“阅读中 / 待复盘 / 已整理”这类状态，用户更需要快速识别，而不是阅读一整张说明卡。

2. 建议动作和动作卡重复

- “建议动作”里显示“本书阅读指南”。
- 下方动作区也有“本书阅读指南”卡片。
- 同一个动作出现两次，会削弱用户对主路径的判断。

3. 待复盘 / 已整理像两张空卡

- `book-state-options` 当前是两张大按钮卡。
- 每张卡只有标题和短描述，信息密度不够。
- 这更适合做成状态切换控件，而不是并列内容卡。

4. 候选入口权重过高

- `book-candidate-entry` 独占整行。
- “加入候选”是低频动作，不应占据和核心动作同等的视觉面积。
- 当前它更像一个第五个主功能模块，而不是辅助入口。

5. 区域整体层级偏多

- 同一屏里同时出现：区域标题、状态卡、状态选项、四个动作卡、候选入口。
- 每层都用了卡片边框和较大 padding，导致视觉分块过多。
- 用户扫读时会不断重新判断“这里是不是一个新的模块”。

## 改造目标

1. 明确主次

- 第一优先级：当前状态和建议下一步。
- 第二优先级：查看笔记、AI 复盘、找相似、本书阅读指南。
- 第三优先级：待复盘 / 已整理标记、加入候选、清除状态。

2. 提升信息密度

- 减少大块空白。
- 降低低频动作占用面积。
- 保留可读性和可点击面积。

3. 保持现有能力

- 不删除任何操作。
- 不改变本地整理状态逻辑。
- 不改变候选保存逻辑。
- 不改变 AI 复盘、找相似、阅读指南入口。

4. 先做最小改动

- 第一阶段优先改 CSS 和少量 JSX 文案位置。
- 不新增复杂组件。
- 不引入新的设计系统 token。

## 推荐信息结构

建议把“本书管理”收敛为三层：

```text
[本书管理标题]
  本机状态说明

[状态摘要条]
  [阅读中] 继续积累可复盘材料
  当前进度：微信进度 6%
  建议下一步：本书阅读指南

[操作区]
  查看笔记 | AI 复盘 | 找相似 | 本书阅读指南

[辅助操作]
  本地状态：待复盘 / 已整理 / 清除
  加入候选
```

## 具体改造建议

### 1. 状态卡改成摘要条

当前：

- 大卡片。
- 左侧状态 badge。
- 下方两列元信息。
- 底部一段建议说明。

建议：

- 改为更紧凑的横向摘要条。
- 状态 badge、标题、当前进度、建议动作放在同一视觉容器内。
- 下一步说明保持一行短句，避免形成第二个大段落。

目标尺寸：

- `book-asset-status-card` padding 从 `14px` 降到 `12px 14px`。
- gap 从 `12px` 降到 `8px`。
- `book-asset-status-meta` 可从两张卡片改成 inline meta。
- 状态卡高度目标控制在 `120px` 到 `150px`。

### 2. 状态选择改为紧凑分段控件

当前：

- `待复盘` 和 `已整理` 是两张并列大按钮卡。

建议：

- 改为一行紧凑状态选择。
- 标题文案可以保留，但描述文案弱化或移除。
- `清除状态` 保持低权重文本按钮或小图标按钮。

目标：

- `book-state-options` 不再使用 `repeat(auto-fit, minmax(180px, 1fr))` 的大卡片布局。
- 改成 `display: flex; flex-wrap: wrap;` 或三列紧凑 grid。
- 单个状态按钮高度控制在 `34px` 到 `40px`。

### 3. 四个主动作继续保留，但压缩卡片厚度

当前四个动作卡方向正确，不需要推翻。

建议：

- 保持四列。
- padding 从 `12px` 降到 `10px 12px`。
- 图标和文本 gap 从 `10px` 降到 `8px`。
- 描述文本最多两行，避免卡片高度被长文案拉高。
- 动作标题保持 14px 到 15px，不再继续放大。

可考虑调整文案：

- `查看笔记`：确认复盘输入范围。
- `AI 复盘`：整理划线和想法。
- `找相似`：探索同主题书。
- `本书阅读指南`：规划下一步阅读。

### 4. 候选入口降级

当前：

- 候选入口占一整行大卡。

建议：

- 作为辅助操作放到状态选择同一层，或放到动作区末尾的小入口。
- 如果保留整行，视觉上应明显弱于四个主动作。
- 可以改为一行浅色提示：`加入候选：保存到候选书架，用于后续路线和选书决策`，右侧一个小按钮。

目标：

- 不再使用与主动作卡相同的高度。
- 高度控制在 `44px` 到 `56px`。
- 边框可以保留虚线，但背景更淡。

### 5. 去重“建议动作”

建议保留两种方式之一：

方案 A：

- 状态摘要条只显示 `建议下一步：本书阅读指南`。
- 下方动作卡继续保留可点击入口。

方案 B：

- 状态摘要条里的建议动作直接可点击。
- 下方四宫格中移除或弱化“本书阅读指南”。

推荐方案 A。

理由：改动小，用户仍能在动作区找到稳定入口，同时状态区只承担引导，不重复制造一个主按钮。

## 组件落点

### `BookActionPanel`

文件：`src/pages/BookDetailPage.tsx`

建议优先保持函数内部结构，不拆新文件。

可能需要小幅调整：

- `BookAssetStatusCard` 的 DOM 层级。
- `book-state-options` 和 `book-candidate-entry` 的相对位置。
- `ActionButton` 的 description 文案长度。

不建议第一阶段做：

- 把本书管理拆成多个新组件文件。
- 新增状态管理逻辑。
- 新增后端字段。

### 样式

文件：`src/styles.css`

重点选择器：

- `.book-action-panel`
- `.book-action-heading`
- `.book-asset-status-card`
- `.book-asset-status-main`
- `.book-asset-status-meta`
- `.book-asset-status-next`
- `.book-state-options`
- `.book-state-options button`
- `.book-action-grid`
- `.book-action-card`
- `.book-candidate-entry`

## 实施顺序

1. 先收紧 `.book-action-panel` 的整体 gap 和 padding。
2. 将 `.book-asset-status-card` 从厚卡改为紧凑摘要条。
3. 将 `.book-state-options` 改为紧凑状态控件。
4. 压缩 `.book-action-card` 的 padding、gap 和描述行数。
5. 将 `.book-candidate-entry` 降级为辅助入口。
6. 检查“本书阅读指南”重复表达，只保留一个稳定点击入口。
7. 验证桌面和窄屏布局，不出现重叠、溢出和按钮过窄。

## 验收标准

- “本书管理”首屏高度明显下降。
- 用户能先看到当前状态和建议下一步。
- `待复盘 / 已整理` 看起来像状态选择，而不是两张空内容卡。
- `加入候选` 不再抢占整行主模块权重。
- 四个核心动作仍然清晰可点击。
- “本书阅读指南”不再形成强重复。
- 桌面端信息密度提升，移动端不出现横向溢出。
- 所有现有动作仍可触发原有回调。

## 不做范围

- 不重写书籍详情页整体结构。
- 不修改微信读书进度逻辑。
- 不修改本地整理状态的数据模型。
- 不改变候选书保存规则。
- 不新增动画、弹窗或复杂交互。
- 不引入新的图标库或视觉主题。

## 风险

- 状态卡压缩过度后，用户可能看不懂“本机整理”和“微信进度”的边界。
- 状态按钮如果太小，移动端触控体验会下降。
- 候选入口降级后，发现入口可能不如当前明显，需要保留清晰文案。
- 如果移除过多描述，AI 复盘和阅读指南的边界可能变模糊。

## 推荐落地方式

第一阶段采用最小改造：

- 以 CSS 调整为主。
- `BookActionPanel` 只做必要的结构顺序和短文案调整。
- 不拆组件，不改数据。
- 每次改完用现有 `BookDetailPage` 相关测试和一次视觉截图验证。

这符合 KISS 和 YAGNI：先解决当前明确的密度和层级问题，不提前设计复杂的管理工作流。
