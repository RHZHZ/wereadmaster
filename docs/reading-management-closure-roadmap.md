# 阅读管理闭环路线图

## 目标

把当前已经可用的微信读书本地管理工具，从“能看数据、能生成结果”进一步收口为“打开应用就知道下一步做什么”。后续新增功能必须落到三类价值之一：行动、决策或交付物。

当前判断：

- 核心闭环已成立：书架、候选、详情、笔记、统计、复盘、发现、设置都已经能串起来使用。
- 产品闭环还需收口：总览的主动作优先级、统计总计模式、非电子书边界、关键空态和错误恢复仍需要打磨。
- 不继续横向堆 AI 功能：泛聊天、全书问答、通用知识库和纯展示型图表暂不进入开发范围。

## P0：必须补齐

### 1. 总览页主动作收口（已完成）

目标：用户打开应用后，优先看到 3-5 个最值得处理的动作，而不是先浏览多个信息区。

范围：

- 统一继续读、待复盘、本地候选、选书决策、阅读指南、统计建议的排序。
- 同一本书或同一动作只出现一次，避免总览、队列和推荐区重复提醒。
- 无缓存、无笔记、无候选、无 AI 结果时，显示一个明确主动作。
- 只读取本地缓存、本地状态和用户已触发生成的结果，不新增后台同步或自动 AI 调用。

验收标准：

- 首屏能在不滚动的情况下看到最高优先级动作。
- `今日可做` 不超过 5 条。
- 每条动作都能直接跳到对应页面或触发明确本地操作。
- 没有数据时，页面给出同步、保存候选、生成复盘或打开设置等单一主路径。

建议测试点：

- E2E：有继续读、待复盘、候选书和统计建议时，只展示优先级最高的少量动作。
- E2E：候选为空时，总览提示去发现页保存候选。
- E2E：无凭据时，总览只引导设置，不展示需要远端数据的动作。
- 单元测试：动作排序函数覆盖重复书籍、空输入、已读完、已生成复盘等场景。

已完成：

- 今日可做按书籍主体去重：同一本书同时出现在继续读和待复盘时，只保留一个主动作。
- 去重后会顺延到下一条可用复盘或候选动作，避免动作数量被重复主体挤占。
- 已补 E2E 覆盖 `总览今日可做同一本书只保留一个主动作`。
- 无凭据时，今日可做只保留“先连接微信读书”主动作。
- 无书架缓存时，今日可做只保留“同步书架缓存”主动作，笔记同步和候选处理下沉到对应页面空态。
- 已扩展 E2E 覆盖首次启动和空态总览主动作。
- 无本地候选时，今日可做显示“去发现页保存候选”，候选队列空态同步引导去发现页。
- 已补 E2E 覆盖 `总览没有候选书时只给发现页保存候选主路径`。
- 无待复盘信号时，今日可做显示“去笔记中心同步笔记”，不再同时展示“整理最近笔记”和“查看书籍复盘”。
- 已补 E2E 覆盖 `总览没有复盘信号时只给笔记中心同步主路径`。
- AI Provider 未配置时，今日可做不再展示“查看书籍复盘”，改为单一“配置 AI Provider”入口并打开设置。
- 已补 E2E 覆盖 `总览 AI 未配置时只给配置 Provider 主路径`。
- 今日可做改为加权动作池：继续读、待复盘、已生成选书决策、AI 设置、统计建议、候选处理和 AI 复盘入口统一排序、去重并截断到 5 条。
- 已生成选书决策会替代对应候选书的普通候选动作，避免同一本候选书重复提醒。
- 已缓存统计复盘的下一步建议会进入今日可做，点击进入阅读报告页，不自动重新生成。
- 无最近阅读记录但有电子书时，今日可做提供“打开《书名》阅读指南”入口，直接进入本书阅读指南页且只读取本地缓存。
- 已补 E2E 覆盖 `总览将统计建议和已生成选书决策纳入动作排序`、`总览无最近阅读时提供本书阅读指南入口`。

### 2. 统计页总计模式重构（已完成）

目标：总计模式不再像放大版周期报告，而是表达长期阅读资产。

范围：

- 总计模式弱化日均、环比等短周期指标。
- 强化累计时长、长期常读作者、长期分类投入、年度高峰、长读书目。
- 文案使用“累计资产”“长期投入”“代表方向”，避免“本周期节奏”。
- 周、月、年模式保留当前趋势和节奏表达。

验收标准：

- 进入总计模式时，不展示无意义的环比或短周期解释。
- 长期作者、分类和长读书目在总计模式下有明确资产感。
- 总计模式和周/月/年模式在标题、说明和本地解读上有明显语义差异。

建议测试点：

- E2E：切换总计模式后，页面展示“长期”或“累计”语义。
- E2E：总计模式下不出现“本周期”作为主要解读标题。
- 单元测试：统计解读构建函数根据 `mode === "overall"` 走长期资产文案。

已完成：

- 总计模式标题改为“长期阅读资产”，Hero 文案使用“累计资产”“长期投入方向”“代表书目”和“稳定偏好”。
- 总计模式摘要卡改为累计时长、长期阅读天数、代表方向、长读书目，不再展示自然日均和环比。
- 本地统计解读在总计模式下使用长期投入方向、长期代表书目、年度高峰，不再展示“本周期”或“周期变化”。
- 作者偏好、分类偏好、长读书目在总计模式下分别使用“长期常读作者”“长期分类投入”“长期长读书目”。
- 周/月/年模式继续保留现有周期报告、自然日均、环比和周期变化表达。
- 统计页和阅读报告页统一改为 `mode + baseTime` 时间锚点模型：总计可进入历史年份，年度可进入历史月份，月/年/周都支持前后切换历史周期。
- 历史周期标题、时间锚点、Markdown 导出和 AI 提示词统一改为绝对时间表达，避免把历史数据继续写成“本月/今年”。
- 已补 E2E 覆盖 `统计页总计模式表达长期阅读资产`。

### 3. 非电子书边界显式化（已完成）

目标：有声书和文章收藏保持轻管理能力，不让用户误以为它们具备完整书籍详情。

范围：

- 在书架卡片和相关入口中明确区分电子书、有声书、文章收藏。
- 非电子书只提供轻动作：复制标题、去发现页搜索、保存候选或打开微信读书深链。
- 不为有声书和文章收藏伪造章节、笔记或完整详情页。
- 如果某入口暂不支持非电子书，按钮文案和禁用原因必须明确。

验收标准：

- 用户点击非电子书卡片时不会进入空白详情或半成品详情。
- 所有非电子书动作都能完成，或给出明确不可用原因。
- 候选保存时保留类型边界，不写回微信读书远端。

建议测试点：

- E2E：有声书卡片菜单包含复制标题和去发现页搜索。
- E2E：文章收藏不进入电子书详情页。
- E2E：非电子书保存本地候选后，候选书架能显示来源和类型。

已完成：

- 书架中的有声书和文章收藏保持非详情卡片形态，只通过更多菜单提供轻管理动作。
- 非电子书菜单支持复制标题、去发现页搜索和保存候选；保存候选只写入本地状态，不写回微信读书远端。
- 有声书保存为 `itemType: "album"`，文章收藏保存为 `itemType: "mp"`，不伪装成发现页电子书候选。
- 候选书架纳入 `album/mp` 类型的本地候选，并展示“有声书 · 轻管理候选”或“文章收藏 · 轻管理候选”来源。
- 非电子书候选在候选书架不进入完整书籍详情，避免伪造章节、笔记或完整详情能力。
- 已补 E2E 覆盖 `桌面端主流程可导航并使用本地命令 mock 数据` 中的非电子书保存候选、类型展示和文章收藏不进详情。
- 已复核 E2E 覆盖 `书籍详情页可以把发现页书籍加入候选`、`已读完成书籍详情页不提供加入候选入口`，确保电子书详情候选入口边界仍成立。

### 4. 核心空态和错误态收口（已完成）

目标：同步失败、无缓存、无候选、无复盘、AI 未配置时，用户仍知道下一步。

范围：

- 总览、统计、候选书架、复盘中心、阅读指南都提供单一主动作。
- 错误提示包含恢复路径，例如打开设置、重试同步、改用本地缓存、返回候选书架。
- AI 相关错误不暴露密钥、请求体、数据库路径或原始 provider 响应。
- 空态不输出伪结论，不用占位数据假装有洞察。

验收标准：

- 每个核心页面在空数据状态下都有可执行主按钮。
- 每个失败状态都能解释失败来源，并给出下一步。
- AI 未配置时，所有生成入口只引导设置，不自动重试。

建议测试点：

- E2E：无凭据启动后，核心页面均能引导设置。
- E2E：统计无缓存时显示同步入口，不展示伪统计。
- E2E：候选为空时，选书决策入口不可用并引导发现页。
- E2E：AI Provider 未配置时，生成按钮提示配置入口。

已完成：

- 总览空数据时只保留“同步书架缓存”主动作，不同时暴露笔记、候选和复盘的次级入口。
- 书架、笔记、统计、书籍复盘、阅读报告和阅读指南输入范围都提供明确空态主按钮。
- 阅读报告统计空态新增卡片内“同步统计”入口，不再只依赖 Hero 区按钮。
- 候选书架空态明确说明“选书决策需要先保存至少 1 本候选”，并只提供“去发现页保存候选”主路径；没有候选时不展示“推荐下一本”。
- 统计同步失败时错误态显示失败来源，并在错误提示内提供“重试同步”恢复路径。
- AI Provider 未配置时，总览和生成页继续引导设置，不自动重试或自动生成。
- 已补 E2E 覆盖 `空态提供明确下一步动作`、`统计同步失败时错误态提供重试路径`。

## P1：体验增强

### 1. 总览页接入更强动作依据

目标：在不增加自动请求的前提下，让总览动作更像“阅读指挥台”。

范围：

- 接入已生成的下周期建议。
- 接入本地月度阅读画像。
- 接入候选书架决策状态。
- 接入近期打开和待复盘信号。

验收标准：

- 总览动作能解释来自哪个本地信号。
- 所有信号缺失时仍能降级为同步、候选和复盘入口。

### 2. 统计页长期资产补充

目标：让统计页在总计模式下更适合回顾长期阅读资产。

详细方案见 `docs/reading-stats-visualization-refactor-plan.md`，用于统一统计页与阅读报告页的图表策略、页面分层、组件拆分和分阶段施工顺序。

范围：

- 年度阶段变化。
- 长期代表主题。
- 常读作者稳定性。
- 长读内容沉淀。

验收标准：

- 新模块能服务复盘或决策，不只增加图表数量。
- 总计模式下页面不会被单一长列表撑开。

### 3. 阅读指南和选书决策联动

目标：让“当前书怎么读”和“下一本读什么”之间有更自然的跳转。

范围：

- 从阅读指南页进入候选书选择。
- 从选书决策结果回看当前书阅读处方。
- 两个功能继续保持手动触发和本地缓存边界。

验收标准：

- 不出现自动生成阅读指南或自动生成选书决策。
- 跳转时能保留当前书、候选书和输入范围说明。

### 4. 输出物体系收口

目标：让单本笔记、复盘、指南、批量导出和行动清单的 Markdown 更稳定。

范围：

- 统一导出 front matter。
- 统一来源统计说明。
- 统一行动项和完成标准结构。
- 保留每类导出的业务差异，不做过度抽象。
- 统一导出失败展示：错误分类、原始错误、恢复建议、上下文保留和重试入口一致。

验收标准：

- 导出内容不包含 API Key、数据库路径、原始接口响应或敏感日志。
- 导出文件能被外部 Markdown 工具直接归档。
- 导出失败不会丢失用户已选择的范围和设置，也不会静默请求远端或自动调用 AI。

## 暂不做

- 泛 AI 聊天。
- 全书内容问答。
- 通用知识库或向量检索。
- 自动后台 AI 生成。
- 为填补页面空白而增加纯展示图表。
- 需要用户长期维护复杂字段的任务管理系统。

这些方向会显著增加数据边界、引用策略、防幻觉和状态同步复杂度。当前阶段应优先把已有闭环做稳。

## 推荐执行顺序

1. 总览页主动作收口。
2. 核心空态和错误态收口。
3. 统计页总计模式重构。
4. 非电子书边界显式化。
5. P1 中的总览动作依据和输出物体系收口。

这个顺序优先处理用户每天打开应用时最先遇到的问题，再处理报告表达和资产沉淀。

## 第三阶段：版本连续性收口

第三阶段规划已独立收口到 [third-stage-plan.md](./third-stage-plan.md)。

当前只保留三个重点：

- 跨书路线真正接续到上一版资产链。
- 更新时说明清楚基于什么变化。
- 评估书籍复盘是否并入同一套版本体系。

补充发布要求：

- 桌面端发布优先采用 GitHub Releases 作为更新源，支持用户在应用内执行一键检查并安装更新。
- 发布流程需要同时覆盖安装包、更新描述文件和签名产物，不把“发布构建”和“更新分发”拆成两套互不关联的流程。
- 一键更新只面向正式发布版本，不作为开发调试路径。

这一阶段不再新增入口，不再扩主列表，不再把历史结果做成时间线。
