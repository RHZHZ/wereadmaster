# 批量导出向导布局约定

## 目标

批量导出向导优先服务“看清并选择书籍”。预检统计、导出策略和设置项必须辅助决策，不能成为主要高度占用。

笔记页批量导出的主任务是导出划线/想法 Markdown。已生成书籍复盘可以作为随书籍附带导出的兼容能力，但复盘专属批量导出应归属书籍复盘页。

## 桌面布局

- 弹窗外层固定高度，外层不滚动。
- 书籍列表区独立滚动，底部操作区固定可见。
- 预检统计卡使用紧凑高度，只展示关键数字：可直接导出、需要同步、无可导出、已有复盘。
- 三个导出策略保持同一行展示，描述最多承担辅助说明，不应撑高整块设置区。
- “只导出选中的书”策略下，设置工具栏为一行：`排除无划线/想法的书`、搜索框、`已选择 N 本`。
- “先同步缺失笔记再导出”策略下，设置工具栏为一行：同步并发、`排除无划线/想法的书`。
- “仅导出本地已缓存内容”策略下，设置工具栏只保留 `排除无划线/想法的书`。

## 过滤规则

- `排除无划线/想法的书` 默认开启。
- 开启时，只处理可能导出 Markdown 的书；只有书签的书默认不进入一键导出队列。
- 已有 AI 复盘缓存的书即使没有划线/想法也保留，因为仍有可导出的复盘缓存。
- 关闭后，无划线/想法的书可以显示在预检列表中，但导出结果应明确记录为无内容或跳过。
- 同步完成但仍没有划线/想法时，导出结果记为跳过，不按失败处理。

## 复盘导出边界

- 批量导出向导不增加复盘专属筛选、复盘搜索或复盘批量选择。
- 已有复盘缓存可以随同一本书的笔记导出一起写入导出目录。
- 独立“导出复盘”入口应放在书籍复盘页，由该页负责已生成复盘列表、搜索、筛选、批量选择和复盘导出索引。
- 导出复盘只读取已生成缓存，不自动生成缺失复盘。
- 批量生成复盘必须作为单独生成流程设计，不能和导出流程混用。

## 搜索与选择

- 搜索只作用于“只导出选中的书”策略下的当前预检列表，不得替换全局预检数据。
- 默认不全选“只导出选中的书”策略下的列表；用户需要显式勾选。
- `已选择 N 本` 必须和搜索框同排显示，不能掉到下一行。

## 失败与重试

- 单本同步失败属于导出报告中的条目状态，应保留报告并提供单本重试，不阻断其他书籍结果。
- 顶层导出命令失败属于向导结果态，应在弹窗内展示结构化失败信息，不能只依赖 toast。
- 顶层失败信息必须包含错误分类标题、原始错误、恢复建议和状态保留说明。
- 顶层失败后保留预检结果、导出策略、并发设置、过滤设置和已选书籍；用户可以直接重试，也可以返回设置调整范围。
- 错误结果态不得展示进行中状态，避免用户误判导出仍在运行。
- 失败重试不能静默同步微信读书远端，除非用户此前已经选择“先同步缺失笔记再导出”策略。
- 批量导出不会自动生成 AI 复盘；已有复盘缓存只作为随书导出的本地资产。

## 响应式约束

- 窄屏下工具栏可以纵向堆叠。
- 窄屏不能出现水平溢出。
- 即使窄屏堆叠，书籍列表仍应保留独立滚动区域，避免操作按钮或设置项吞掉列表空间。

## 回归验证

- 批量导出 E2E 应覆盖列表独立滚动和底部操作区固定可见。
- 桌面宽度下，书籍列表需要有最小可视高度，避免被设置区挤压。
- 桌面宽度下，选中策略工具栏需要验证过滤、搜索、已选择三项同排显示。
- 批量导出 E2E 应覆盖顶层命令失败后的分类提示、返回设置和重试成功。
- 构建和前端测试必须通过：`npm test`、`npm run build`、`npm run e2e -- --grep "批量导出"`。
